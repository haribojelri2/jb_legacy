## 4. AI Agent 기초 (팀 공유용)

### 한 줄 정의

### 우리가 실제로 구현한 것

| 요소 | 역할 | JB Legacy 적용 |
|---|---|---|
| **LLM (두뇌)** | 판단·계획·자연어 | GPT-4o / GPT-4o-mini |
| **Tools (손·발)** | 실제 계산 함수 | `calc_goodwill_tax()`, `estimate_business_value()`, `design_retirement_cashflow()` |

| **Planning Loop** | 다단계 의사결정 | LangGraph 순차·조건부 실행 |

## 5. 멀티 에이전트 아키텍처

### 4 Core Agents

| 에이전트 | 역할 | 핵심 도구 |
|---|---|---|
| **Business Valuation Agent** | 사업체 권리금·적정가치 산정 | `estimate_business_value()` — 월순이익 × 연수별 배수(24~36개월) |
| **Tax & Succession Agent** | 매각 vs 승계 세금 RAG 시뮬 | ChromaDB RAG + `calc_goodwill_tax()` + `calc_gift_tax_special()` |
| **Post-Exit WM Agent** | 은퇴 후 30년 현금흐름 설계 | `design_retirement_cashflow()` + JB자산운용·전북은행 상품 연계 |
| **Family Bridge Agent** | 부모↔자녀 승계 리포트 공유 | 이과장 앱으로 시뮬레이션 결과 발송 |

+ **Compliance Guard**: 세무 면책·금소법 자동 검수 (Reflexion 패턴, max 3회 재시도)
+ **Slow UI Adapter**: 62세 자영업자 대상 어르신 모드 변환

### 아키텍처 다이어그램

```
              [User Query]
                   ↓
             [Supervisor]
            라우팅 결정
       (tax / post_exit / family / valuation)
                   ↓
             [Profiler]
            이사장·이과장 프로필 로드
                   ↓
     [Business Valuation Agent]
      권리금·사업가치 산정
                   ↓
     [Tax & Succession Agent]  ← ChromaDB RAG (세법 문서)
      매각세금 vs 승계세금 비교
                   ↓
     [Post-Exit WM Agent]
      8억 → 월 420만원 포트폴리오
                   ↓
     [Slow UI Adapter]
      최종 응답 생성 (Slow/Normal)
                   ↓
     [Compliance Guard]
      세무 면책·금소법 검수
          ↓ (통과?)
          ├─ 재생성 (max 3회)
          └─ [Family Bridge] → END
```

### RAG 구조 (실제 구현)

```
국세청 공식 문서 5개
  ├─ 가업승계 증여세 과세특례 (조특법 30조의6)
  ├─ 가업상속공제 (상증법 18조의2)
  ├─ 권리금 기타소득 과세 (소득세법)
  ├─ 2025년 종합소득세·양도소득세 세율표
  └─ 은퇴 후 JB금융 연계 포트폴리오
         ↓ (OpenAI text-embedding-3-small)
     ChromaDB 벡터 스토어
         ↓ (similarity_search)
     쿼리 관련 문서 청크 → LLM 컨텍스트
```

## 6. 발표용 데모 시나리오 (3 Scenes)

### Scene 1 (2분) — 매각 vs 승계 세금 디베이트
이사장: *"딸한테 물려주는 게 나을까, 그냥 권리금 받고 파는 게 나을까?"*

→ Tax & Succession Agent가 RAG로 조특법 30조의6 검색 후 실제 계산:
- **외부 매각**: 권리금 3억 × 40% = 과세표준 1.2억 → 세금 약 **4,700만원**
- **가업승계 과세특례**: 사업가치 4.5억 - 10억 공제 = **세금 0원**
- UI에 두 카드를 나란히 표시 → 절세 효과 시각화

**시각적 강조**: 세금 비교 카드 + "RAG 참조 세법 문서" 패널

### Scene 2 (2분) — 가족 협업 모드 (Split-Screen)
이사장: *"서울에 있는 딸한테 내 은퇴 계획 알려줄 수 있어?"*

→ Family Bridge Agent 발동 → 이과장 앱에 리포트 도착
- 왼쪽 화면: 이사장 (Slow Banking UI — 큰 글씨, 쉬운 설명)
- 오른쪽 화면: 이과장 (요약 카드 — 세금 비교표, 월 수령액, 메시지)

**시각적 강조**: 부모/자녀 split-screen 동시 표시

### Scene 3 (2분) — 엑시트 후 평생 WM 설계
이과장: *"아버지 식당 팔고 8억 생기면 노후 생활비 충분할까?"*

→ Post-Exit WM Agent:
- 3억 → JB자산운용 월배당 펀드 → 월 125만원
- 3억 → 전북은행 정기예금 (연 3.55%) → 월 89만원
- 2억 → 개인연금
- 국민연금 월 90만원
- **합계: 월 약 420만원** (목표 300만원 대비 +120만원 여유)

## 7. 세금 계산 로직 (국세청 공식 기준)

### 권리금 기타소득세 계산식
```
과세표준 = 권리금 × 40%  (필요경비 60% 인정, 소득세법)
세금 = 과세표준 × 종합소득세율 (누진세) - 누진공제
지방소득세 = 세금 × 10%
```

### 2025년 종합소득세 누진 구간 (국세청 공식)
| 과세표준 | 세율 | 누진공제 |
|---|---|---|
| 1,400만원 이하 | 6% | — |
| ~5,000만원 | 15% | 126만원 |
| ~8,800만원 | 24% | 576만원 |
| ~1억 5천만원 | 35% | 1,544만원 |
| ~3억원 | 38% | 1,994만원 |
| ~5억원 | 40% | 2,594만원 |
| ~10억원 | 42% | 3,594만원 |
| 10억원 초과 | 45% | 6,594만원 |

### 가업승계 증여세 과세특례 (조특법 30조의6)
- 10억원 공제 후 **10% 단일세율** (60억 초과분 20%)
- 최대 600억 한도
- 요건: 10년 이상 경영, 60세 이상 증여자, 18세 이상 수증자
- 사후관리: 5년간 가업 유지, 지분 감소 금지

## 8. 왜 이 주제가 심사위원에게 먹히는가

### 완벽한 JB 핏
전북·전남 지방 자영업자 비중은 전국 평균 대비 높음. JB우리캐피탈 사업자 여신 데이터 + 전북은행 급여통장 데이터 → 실제 구현 가능한 근거 제시 가능.

### 경쟁 공백이 실재함
KB/신한 WM AI는 "이미 자산가인 사람"의 포트폴리오 최적화. **사업체가 전 재산인 5060 자영업자의 출구 전략**을 다루는 AI는 없음. 시중은행이 안 하는 이유: 고객층이 지방·소규모이기 때문 → JB의 강점이 그대로 차별화.

### 완벽한 크로스셀링(Cross-selling) 비즈니스 모델: C안(절충안)의 가치
단순한 컨설팅을 넘어, 은퇴하는 자영업자(부모)와 가업을 승계받는 자녀 사이의 '자금 공백'을 JB금융그룹의 대출로 메우는 구조(C안: 절충안)를 제안합니다.
- 자녀는 **JB우리캐피탈**에서 승계 자금 대출을 받아 부모에게 권리금 일부를 현금으로 줍니다. (**여신 발생**)
- 부모는 그 현금을 **전북은행 예금 및 JB자산운용 펀드**에 예치하여 노후 생활비를 만듭니다. (**수신 발생**)
- 1건의 AI 상담으로 부모와 자녀 모두를 고객으로 유치하고 여신/수신 양쪽에서 수익을 창출하는, **실제 은행업의 본질에 가장 부합하는 기획**입니다.

### 이종 데이터 연계 (공식 요구사항 충족)
- 사업 데이터 (POS 매출, 급여통장) + 세제 데이터 (RAG) + 개인 자산 데이터 → 통합 분석
- 공식 문구 "이종 데이터 연계" 100% 충족

### 가족 보호자 알림 (공식 요구사항 충족)
부모↔자녀 Family Bridge → "가족·보호자 알림" 공식 요구사항 그대로 충족

### 감정적 임팩트
"평생 일군 가게" — 숫자가 아니라 사람 이야기. 심사위원 중 자영업 부모 있는 사람이 기억에 남길 장면.
