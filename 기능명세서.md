# JB Legacy — 기능명세서

**서비스명:** JB Legacy — 자영업자 AI 은퇴 플래닝 어시스턴트 (지정주제 1, MVP v1.1)

---

## 1. 서비스 개요 (Service)

전북·광주 지역 50~65세 자영업자가 은퇴 시점에 직면하는 사업체 처분 의사결정(매각·가업승계·절충·가족 합의안)을 AI 멀티에이전트가 세금 시뮬레이션, 노후 포트폴리오, 은퇴 자산 생존 확률, 가족 협상, 이상거래 보호까지 통합 지원한다.

본 시스템은 상황과 데이터를 이해하고 판단하며 적절한 행동을 자율 선택하고 결과를 검증·개선하는 Agent형 구조를 갖는다.

| 단계 | 구현 |
|---|---|
| 판단 | 투입 에이전트 결정(Supervisor), 필수 정보 충족 판단(Profiler), 시나리오 권고 A/B/C/D 중 1개 강제(Synthesizer), 이상거래 위험도 분류(FraudGuard) |
| 행동 | 추가 질문, 근거 조회(세법 RAG), 수정안 생성(검수 반영 재생성), 알림(가족 리포트·이상거래 통지), 요약(종합 의견·시니어 변환·음성 브리핑), 예약(PB·세무사) |
| 검증/개선 | 이중 검증 루프 — (1) ComplianceGuard 검수 미통과 시 재생성(최대 3회 종료 조건, 초과 시 PB 연결), (2) 적대 검증 GANReview(공격자·방어자·심판 토론 채점, GAN_AUTO 모드)가 '재생성필요' 판정 시 개선 지시 주입 재생성(최대 1회). 생존 확률 목표(85%) 미달 시 지속가능 생활비 자동 역산. 규칙 기반 계산기와 단위 테스트로 수치 상시 검증 |

## 2. 시스템 구성도 (Architecture)

사용자 입력(Streamlit UI, 텍스트 또는 음성 STT)이 LangGraph 오케스트레이터로 전달되어 다음 순서로 처리된다.

| 순서 | 노드 | 역할 | 분기 |
|---|---|---|---|
| 1 | Supervisor | 질문 분석, 투입 에이전트 동적 선택 | 항상 |
| 2 | Profiler | 프로필 로드, 필수 정보 확인 | 정보 누락 시 추가 질문 후 종료 |
| 3 | 병렬 fan-out | BusinessValuation · TaxSuccession · PostExitWM · Negotiation 동시 실행 | 각 노드가 선택 여부 자체 판단 |
| 4 | Synthesizer | 결과 종합, 시나리오 권고(A/B/C/D 1개) | 항상 |
| 5 | SlowUIAdapter | 시니어용 쉬운 표현 변환 | 시니어 모드만 |
| 6 | ComplianceGuard | 금소법·세무 면책 검수 | 미통과 시 4번으로 재시도(최대 3회) |
| 7 | GANReview | 적대 검증(공격자·방어자·심판 채점) | GAN_AUTO 모드에서 실행, '재생성필요' 시 4번으로 1회 재시도 |
| 8 | FamilyBridge / Booking | 자녀 리포트 발송 / 상담 예약 | 키워드 감지 시 |

- 대화 상태는 SQLite 체크포인터(jb_legacy_memory.db)에 thread_id 단위로 영속 저장되어 후속 질문·what-if 재분석에서 맥락이 유지된다.
- 병렬 노드는 서로의 중간 결과를 참조할 수 없으므로, 타 노드 산출이 필요하면 공용 계산 모듈(tools)을 직접 호출해 수치 일관성을 보장한다.
- 거래 안심(FraudGuard)·조기경보·권리금 동적평가·청년 매칭·PDF 리포트·음성 브리핑은 UI 계층 기능으로 분석 패널에 제공된다.

## 3. 핵심 기능 명세 (Feature Specification)

### 3-1. 에이전트별 명세

| 에이전트 | 역할 | 구성·핵심 로직 |
|---|---|---|
| Supervisor | 질문 분석·라우팅 | Claude(fast tier), Pydantic 구조화 출력. 핵심 3개 미선택 시 기본 투입, 가족·협상은 (명사 AND 동사) 결정론 규칙 우선 |
| Profiler | 프로필 로드·추가 질문 | 필수 항목(월 순이익·운영 기간·권리금) 누락 시 1개 질문 생성, 시니어 모드 결정 |
| BusinessValuation | 사업체 가치평가 | 수익환원법: 권리금 = 월 순이익 x 업력 배수(10년 미만 24·20년 미만 30·20년 이상 36개월) + 보증금 + 시설 |
| TaxSuccession | 세금 3시나리오 | Rule Engine + RAG(2025 개정 세법). 매각(권리금 40% 과세표준 종합소득세), 가업승계 특례(조특법 30조의6, 10억 공제 후 10%), 일반 증여(5천만 공제 누진) |
| PostExitWM | 포트폴리오·생존 확률 | 전북은행·광주은행 실제 상품(JB 리치100 즉시연금형·KJ 정기예금·IRP)을 목표 생활비 기반 동적 배분, 세후 산출, 주택연금 가산, 몬테카를로 생존 확률 |
| Negotiation | 가족 합의안(D안) | 4번째 병렬 노드. 자녀 협상 조건(폼) 또는 대화 자동 추출(기본 70%/20%)로 D안 산출, 결정론 게이트로 일반 질문 차단 |
| Synthesizer | 종합·권고 | Claude(smart tier). 3축(부모 안정·가족 자산·균형) 분석, A/B/C/D 1개를 구조화 출력으로 강제, 계산 근거는 코드가 직접 조립, 마크다운 새니타이저로 출력 형식 코드 보장 |
| ComplianceGuard | 금소법 검수 | Claude(smart tier), Pydantic 구조화 판정. 5개 규칙(불확실성·세무사 권장·원금손실·책임소재·단정 금지), 미통과 시 재생성 최대 3회 |
| GANReview | 적대 검증 | Critic·Defender·Judge 3-LLM 토론으로 5개 기준(금융정확성·규정준수·논리일관성·수치근거·편향성) 채점. GAN_AUTO=1 시 그래프 내 자동 실행, '재생성필요' 판정 시 개선 지시 주입 재생성(1회). UI 'AI 검증' 탭에서 수동 실행·전후 점수 비교도 지원 |
| FamilyBridge | 가족 리포트 | 가족 키워드 시 자녀 앱으로 결과 요약 발송 |
| Booking | 상담 예약 | 예약 키워드 시 지점·일시·준비 서류 안내 생성 |
| FraudGuard | 거래 안심 | 룰 4종(고액 이탈 평균+3σ·심야·동일가맹점 분할·해외), 복수 사유=위험 높음, 소비흐름 분석, 가족 알림. LLM 미사용 |
| YouthMatching | 청년 창업가 매칭 | 매각(A·C안) 시 외부 인수자 매칭. 업종·지역·신뢰도 점수로 후보 선정 후 자금 충족성(권리금 대비 자기자본+대출한도) 검증. LLM 미사용, JB 인수 대출 상품 안내 포함 (3-4 참조) |

### 3-2. 세금 계산 (국세청 공식 세율)

- 외부 매각: 권리금의 40%를 과세표준 산입(필요경비 60% 인정), 사업소득 합산 종합소득세 누진(6~45%), 지방소득세 10% 가산
- 가업승계 과세특례: 10억원 공제 후 10%(과세표준 60억 초과분 20%), 한도 600억, 사후관리 요건 안내
- 계산용·표시용 세율표는 단일 모듈(tools/calculators.py) 공통 참조로 불일치를 구조적으로 차단, 대표 수치는 단위 테스트로 고정

### 3-3. 은퇴 자산 생존 확률 (몬테카를로)

- 자산 수익률 월 단위 정규분포 + 긴급 의료비 복합 포아송 과정(연 0.25회, 1회 평균 3,000만원 지수분포)
- 1,000회 경로 100세까지, 난수 시드 고정으로 재현성 확보, 모든 시나리오에 공통 난수 적용으로 순수 비교 보장
- 순인출 = 목표 생활비 − (국민연금 + 주택연금 + 자문료, 자문료 10년 한정). 생활비·국민연금은 연 2% 물가연동 에스컬레이션, 주택연금(정액형)·자문료(계약 고정)는 명목 고정. 출력: 생존 확률 곡선, 고갈 연령 분포, 잔여 자산 중앙값·하위 10%

### 3-4. 청년 창업가 매칭 (외부 인수 생태계)

매각(A안)·절충(C안)처럼 가게를 외부에 넘기는 시나리오에서, 폐업 대신 지역 청년 창업가에게 인수시켜 상권·고용을 유지하는 출구를 제안한다.

- **후보 선정 점수** = 업종 겹침 × 2 + 지역 일치(1/0) + 청년 신뢰등급 × 0.5. 업종·지역이 모두 불일치인 후보는 제외하고 상위 3명 반환. 설계 의도는 "업종 적합성 최우선 + 지역 연고 + 검증된 신뢰도" 순.
- **인수 자금 = JB 인수 대출 조달(신규 여신 고객 창출)**: 권리금을 청년 자기자본으로 다 충당하지 못하는 부족분을 JB 인수 대출로 조달하는 것이 핵심 설계다. 이 대출액(loan_needed = 권리금 − 자기자본, 한도 내)이 곧 JB 신규 여신 창출액(jb_new_credit)이다. 가게는 한 번만 매각되므로 신규 여신은 인수 청년 1명의 대출액이며(합계 아님), 매칭 1순위 인수자 기준 대표값과 인수 가능 후보별 범위를 함께 제시한다. 즉 청년 매칭은 단순 인수자 소개가 아니라 폐업 대신 상권·고용을 유지하면서 JB의 신규 여신 고객을 발굴하는 사업 연계 기능이다.
- **양방향 크로스셀링**: 부모의 매각 대금(수신) + 청년 인수자의 인수 대출(여신)이 JB 안에서 순환. can_afford는 자기자본 또는 대출로 인수 성사 가능한지, funding_gap은 대출 한도로도 부족한 금액을 표시한다.
- **JB 여신 연계**: JB 청년 창업 대출 상품(금리·한도·대상) 안내 + 인수 절차 5단계(의향서 → JB 인수 대출 사전승인 → 계약 → 권리금 수취·세무신고 → 노하우 전수) 제공. 자녀 승계(B·D안)에서는 노출하지 않는다.
- 노출 조건은 LLM 추천 시나리오가 아니라 "매각 시나리오가 분석에 포함되는지"(결정론)로 판정해 안정적으로 표시된다.
- goal-seek 역산: 생존 확률이 목표(85%) 미달인 시나리오는 이분 탐색으로 달성 가능한 지속가능 월 생활비를 자동 산출해 개선안으로 제시(공통 난수 고정 → 결정론 수렴)

### 3-4. 상태 관리

AgentState(TypedDict)가 단일 진실 공급원이며, 병렬 노드가 동시에 쓰는 active_agents는 리듀서로 충돌 없이 누적된다. 모든 LLM 호출은 단일 팩토리(agents/llm.py)를 경유하여 환경변수로 모델 교체가 가능하고 API 키 미설정 시 명확히 중단된다.

## 4. 주요 기능 흐름도 (Flow)

시연 조건: 이사장(62세, 30년 한정식, 월 순이익 450만원), 승계 의향 예, 상권 보합, 은퇴 3년 이내, 목표 300만원, 주택연금 활용

| 단계 | 처리 | 결과 |
|---|---|---|
| 질문 입력 | "가게를 딸한테 넘길지 팔지 모르겠어요" | Supervisor가 3개 핵심 에이전트 투입 |
| 병렬 분석 | 가치평가·세금·자산운용 동시 실행 | 가치 3억 1,200만 / 매각세 2,875만·특례 0원 / 월수령 A 288만·B 289만·C 282만 (전북은행·광주은행 실제 상품, 동적 배분) |
| 종합·검수 | Synthesizer 권고 + 계산 근거 자동 첨부 → ComplianceGuard 검수 | 시나리오 1개 추천, 세무사 권장·원금손실 경고 자동 삽입 |
| 가족 협상 | "딸이 70% 승계에 자문료 20% 제안" | D안 부모 월 292만, 자녀 10년 누적 3억 2,000만 — A/B/C와 동일 기준 비교 |
| 거래 안심 | 최근 90일 거래 점검 | 심야 고액 이체 등 4건 감지, 이과장에게 알림 |
| 마무리 | 가족 공유 / PDF 리포트 / 상담 예약 | 자녀 화면 전달, PB 상담 자료 출력 |

what-if: 대화 중 "상권이 하락세라면" 등 조건 변경을 감지해 재분석하고 변경 전후 수치를 비교 제시한다.

### 6-3. UI 분석 패널

분석 결과(세금 카드·시나리오 포트폴리오·생존 확률 차트·D안·자문료 계약·PDF 리포트·가족 공유), 경영 건강, 권리금 평가, 청년 매칭, 거래 안심(소비흐름·이상거래 경보·가족 알림) 5개 탭. 진행 상태는 노드별 한국어 라벨로 실시간 표시(질문 분석·세금·승계 분석·자산운용 시뮬레이션 등).

### 6-4. 데이터 출처 고지

조기경보·권리금 동적평가·청년 매칭·자문료 계약·거래 안심은 데모 시뮬레이션 데이터로 구현하여 화면에 데모 데이터임을 명기한다(data/mock_mydata.py, data/db.py). 본선에서 마이데이터·FDS·플랫폼 DB 연동으로 교체한다. 거래 안심의 실시간 차단은 은행 고유 FDS 영역이며, 본 서비스는 고객 동의 기반 거래내역 조회를 활용한 보조 감시·가족 통지에 집중한다.
