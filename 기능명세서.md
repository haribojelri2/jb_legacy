# JB Legacy — 기능명세서

**서비스명:** JB Legacy — 자영업자 AI 은퇴 플래닝 어시스턴트  
**버전:** MVP v1.0  
**작성일:** 2026년 5월

---

## 1. 서비스 개요

### 1-1. 목적

전북·광주 지역 50~65세 자영업자가 은퇴 시점에 직면하는 사업체 처분 의사결정(매각 vs 가업승계 vs 절충)을 AI가 세금 시뮬레이션·노후 포트폴리오·가족 협상까지 통합 지원한다.

### 1-2. 시스템 구성도

```
[사용자 입력 — Streamlit UI]
        │
        ▼
┌─────────────────────────────────────────────────────────┐
│                    LangGraph 오케스트레이터               │
│                                                         │
│  [Supervisor]  →  [Profiler]                            │
│       LLM 라우팅       프로필 로드·                       │
│       에이전트 선택     추가 질문 생성                     │
│                          │                              │
│              ┌───────────┤ 병렬 Fan-out                  │
│              ▼           ▼           ▼                  │
│   [BusinessValuation] [TaxSuccession] [PostExitWM]      │
│    수익환원법 가치평가  RAG+세금계산기  3시나리오 포트폴리오  │
│              │           │           │                  │
│              └───────────┴───────────┘                  │
│                          │ Fan-in                       │
│                          ▼                              │
│                   [Synthesizer]                         │
│                   종합 응답 생성                          │
│                          │                              │
│                   [SlowUIAdapter]  ← 느린 UI 모드 변환   │
│                          │                              │
│                   [ComplianceGuard] ← 금소법 자동 검수   │
│                     PASS ↓  FAIL → Synthesizer 재시도   │
│                          ▼                              │
│                   [FamilyBridge]  ← 자녀 앱 리포트 발송  │
│                          │                              │
│                   [Booking]       ← PB·세무사 예약       │
│                          │                              │
│                        [END]                            │
└─────────────────────────────────────────────────────────┘
        │
        ▼
[SQLite 체크포인터 — 대화 상태 영속 저장 (thread_id별)]
```

## 2. AI Agent 상세 설계

### 2-1. Supervisor Agent (질문 분석·에이전트 라우팅)

| 항목 | 내용 |
|---|---|
| **역할** | 사용자 질문 분석 → 필요한 전문 에이전트 목록 동적 결정 |
| **모델** | GPT-4o-mini (temperature=0) |
| **입력** | 사용자 자연어 질문 |
| **출력** | selected_agents 목록 (예: ["BusinessValuation", "TaxSuccession"]) |
| **로직** | 단일 주제 → 해당 에이전트 1개만 선택. 복합 주제 → 복수 선택. 핵심 3개 미선택 시 전부 기본 투입 |
| **특이사항** | "딸/아들/자녀/공유/전달" 키워드 감지 시 FamilyBridge 자동 추가 |

**선택 기준표:**

| 질문 유형 | 투입 에이전트 |
|---|---|
| 종합 상담 ("어떻게 해야 할까") | BusinessValuation + TaxSuccession + PostExitWM |
| 세금·절세·증여 단독 | TaxSuccession |
| 자산운용·포트폴리오·노후 단독 | PostExitWM |
| 권리금·시세·가치평가 단독 | BusinessValuation |
| 가족 공유 요청 | 위 중 해당 + FamilyBridge |

---

### 2-2. Profiler Agent (사용자 프로필 로드·추가 질문)

| 항목 | 내용 |
|---|---|
| **역할** | DB에서 고객 프로필 로드. 핵심 정보 누락 시 1개 추가 질문 생성 |
| **모델** | GPT-4o-mini |
| **필수 확인 항목** | 월 순이익, 운영 기간, 권리금(예상 매도가) |
| **출력** | user_profile dict, clarification_needed (추가 질문 or 빈 문자열) |
| **UI 모드 결정** | 프로필의 slow_ui=True이면 SlowUIAdapter 활성화 |

---

### 2-3. BusinessValuation Agent (사업체 가치평가)

| 항목 | 내용 |
|---|---|
| **역할** | 권리금·보증금·시설·집기를 포함한 총 사업체 가치 산출 |
| **모델** | GPT-4o-mini (설명 생성) + Rule Engine (수치 계산) |
| **계산 방식** | 수익환원법: 월 순이익 × 멀티플(업력 기반, 12~48배) |
| **출력 항목** | 권리금 추정액, 보증금, 시설·집기, 합계, 자영업자 눈높이 설명문 |

**멀티플 기준 (업력별):**

| 업력 | 등급 | 권리금 멀티플 |
|---|---|---|
| 3년 미만 | D | 12개월 |
| 3~10년 | C | 24개월 |
| 10~20년 | B | 30개월 |
| 20~30년 | A | 36개월 |
| 30년 이상 | S | 48개월 |

---

### 2-4. TaxSuccession Agent (세금·승계 시뮬레이션)

| 항목 | 내용 |
|---|---|
| **역할** | 매각·가업승계·일반증여 3시나리오 세금 자동 계산 및 비교 |
| **모델** | GPT-4o (temperature=0) + Rule Engine + RAG |
| **RAG 데이터** | 2025년 개정 세법 (ChromaDB, text-embedding-3-small) |

**3시나리오 계산 내용:**

| 시나리오 | 세목 | 계산 기준 |
|---|---|---|
| **A. 외부 매각** | 권리금 기타소득세 | (권리금 × 70%) × 기타소득세율 (6~45%) |
| **B. 가업승계 과세특례** | 증여세 (조특법 §30의6) | 과세가액 600억 한도, 10% 단일세율 |
| **C. 일반 증여** | 증여세 | 누진세율 10~50%, 기본공제 5천만원 |

---

### 2-5. PostExitWM Agent (은퇴 후 자산운용 시뮬레이션)

| 항목 | 내용 |
|---|---|
| **역할** | 3가지 은퇴 시나리오별 노후 월 수령액·포트폴리오 비교 |
| **모델** | GPT-4o (temperature=0.1) + Rule Engine |
| **상품 기반** | JB자산운용·JB생명·전북은행 실제 상품 카탈로그 (jb_products.py) |

**3시나리오 구성:**

| 시나리오 | 설명 | 포트폴리오 성향 |
|---|---|---|
| **A. 완전 매각** | 사업체 전체 매각, 세후 현금 전액 운용 | 보수·균형 (시장 트렌드·은퇴 시기 반영) |
| **B. 완전 승계** | 자녀에게 전부 이전, 자문료로 소득 보완 | 성장·균형 |
| **C. 절충안** | 권리금 50% 현금화 + 나머지 승계 | 균형 |

**포트폴리오 구성 (4분산):**

| 상품 유형 | 비율 (균형형) | JB 실제 상품 |
|---|---|---|
| 월배당펀드 | 30% | JB자산운용 월배당 펀드 |
| 즉시연금 | 28% | JB생명 즉시연금 |
| 정기예금 | 27% | 전북은행 정기예금 |
| IRP(퇴직연금) | 15% | 전북은행 IRP |

- 모든 수령액은 **세후 기준** (이자·배당소득세 15.4%, 연금소득세 5.5% 차감)
- 주택연금 (HF공사 공시 기준, 종신 정액형) 선택 시 추가 반영
- 0·5·10·15·20년 후 명목·실질 월수령액 장기 시뮬레이션 제공

---

### 2-6. ComplianceGuard Agent (금소법·세무 면책 검수)

| 항목 | 내용 |
|---|---|
| **역할** | 최종 응답이 금소법·세무 면책 기준을 충족하는지 자동 검수 |
| **모델** | GPT-4o-mini (temperature=0) |
| **최대 재시도** | 3회 (실패 시 PB·세무사 연결로 전환) |

**검수 규칙 (5개):**

1. 세금 추정치에 '약', '예상', '추정' 등 불확실성 명시
2. '세무사 상담을 권장합니다' 문구 포함
3. 투자 추천 시 '원금 손실 가능성'과 위험등급 명시
4. '최종 결정은 고객 본인 또는 담당 PB·세무사가 해야 함' 명시
5. 특정 상품을 단정적으로 '최선'이라고 표현 금지

---

### 2-7. FamilyBridge Agent (가족 리포트 공유)

| 항목 | 내용 |
|---|---|
| **역할** | 시뮬레이션 결과를 자녀 JB 앱으로 Push 발송 |
| **트리거** | 질문에 "딸/아들/자녀/공유/전달/보내" 키워드 포함 시 |
| **발송 내용** | 사업체 가치, 시나리오별 세금, 은퇴 후 월 수령액 요약 |

---

### 2-8. Negotiation Agent (가족 협상 AI — 핵심 차별화 기능)

| 항목 | 내용 |
|---|---|
| **역할** | 자녀의 협상 조건 입력 → 부모·자녀 모두에게 유리한 D안(합의안) 자동 산출 |
| **모델** | GPT-4o (temperature=0.1) |
| **입력** | 자녀의 승계 비율(0~100%), 자문료율(순이익 대비 %), 자녀의 한마디 |
| **출력** | 부모 월 수령합계, 자녀 10년 누적 수익, 10년 후 권리금 추정, 합의안 설명문 |

**협상 D안 계산 흐름:**

```
자녀 승계 비율 입력 (예: 70%)
        │
        ▼
부모 현금화 비율 = (1 - 70%) × 권리금
        │
        ├─ 권리금 기타소득세 차감
        ├─ 개인 저축 합산
        └─ 월배당펀드·즉시연금·IRP·정기예금 포트폴리오 구성
        │
자녀 자문료 = 순이익 × 자문료율 (10년간 부모 지급)
        │
        ▼
가업 지속 가치 계산
  ├─ 딸 10년 누적 수익 (영업이익 × 상권트렌드 × 승계비율)
  └─ 10년 후 권리금 추정 (현재가치 × 성장율^10)
```

### 2-9. Booking Agent (PB·세무사 상담 예약)

| 항목 | 내용 |
|---|---|
| **역할** | 시뮬레이션 완료 후 PB 또는 세무사 상담 예약 자동 연결 |

## 3. 데이터 흐름 및 상태 관리

### 3-1. AgentState 구조

```python
AgentState = {
    # 공통
    "messages":             list[dict],        # 대화 이력
    "user_id":              str,               # 사용자 식별자
    "query":                str,               # 현재 질문
    "selected_agents":      list[str],         # Supervisor 선택 에이전트
    "active_agents":        list[str],         # 실행된 에이전트 이력

    # 각 에이전트 출력
    "user_profile":         dict,              # Profiler
    "business_valuation":   dict,              # BusinessValuation
    "tax_comparison":       dict,              # TaxSuccession (sale/special/general)
    "retirement_portfolio": dict,              # PostExitWM (scenario A/B/C)
    "negotiation_result":   dict,              # Negotiation (D안)
    "family_notified":      bool,              # FamilyBridge
    "compliance_passed":    bool,              # ComplianceGuard
    "booking_result":       dict,              # Booking

    # 제어
    "clarification_needed": str,               # 추가 질문 (있을 경우)
    "retry_count":          int,               # 컴플라이언스 재시도 횟수
    "final_response":       str,               # 최종 출력 텍스트
    "ui_mode":              str,               # "normal" | "slow"
}
```

### 3-2. 대화 영속성

- SQLite 체크포인터 (`jb_legacy_memory.db`)로 thread_id별 대화 상태 전체 저장
- 사용자가 이어서 질문해도 이전 프로필·시뮬레이션 결과가 유지된다

## 6. 핵심 시나리오 시연 흐름

### 시나리오: "이사장이 딸에게 가게를 넘길지 팔지 모르겠다"

```
1. 이사장 선택 → 삶의 조건 설정 (승계 의향: 예, 상권: 보합, 은퇴: 3년 이내)
2. 질문 입력: "가게를 딸한테 넘겨야 할지 팔아야 할지 모르겠어요"

3. [질문 분석] Supervisor: BusinessValuation + TaxSuccession + PostExitWM 선택
4. [프로필 로딩] Profiler: 이사장 프로필 로드 (월순이익 450만원, 30년 운영)
5. [병렬 실행]
   ├─ [사업체 가치 평가] 총 3.1억원 (권리금 1.62억 + 보증금 1억 + 시설 5천만)
   ├─ [세금·승계 분석]
   │   ├─ A안 매각: 세금 약 3,200만원
   │   ├─ B안 가업승계 특례: 세금 약 1,620만원 (약 1,600만원 절세)
   │   └─ C안 일반증여: 세금 약 4,100만원
   └─ [자산운용 시뮬레이션]
       ├─ A안 (완전매각): 세후 운용자산 2.8억 → 월 수령 약 290만원
       ├─ B안 (완전승계): 자문료 포함 → 월 수령 약 180만원
       └─ C안 (절충): 1.5억 운용 + 자문료 → 월 수령 약 240만원
6. [종합 의견 생성] 3시나리오 비교 설명 + 이사장 상황에 맞는 제안
7. [금소법 검수] 세무사 권장·원금손실 경고 등 자동 삽입·검수 통과
8. [가족 리포트 공유] 이과장 앱으로 리포트 발송

9. [협상 탭] 이과장이 입력: 승계 70%, 자문료 20%, "아빠 걱정 마세요"
   → D안: 부모 월 수령 약 255만원, 딸 10년 누적 수익 약 4.1억원
```
