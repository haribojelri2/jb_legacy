# JB Legacy — 기능명세서

**서비스명:** JB Legacy — 자영업자 AI 은퇴 플래닝 어시스턴트
**버전:** MVP v1.1
**작성일:** 2026년 6월

---

## 1. 서비스 개요

### 1-1. 목적

전북·광주 지역 50~65세 자영업자가 은퇴 시점에 직면하는 사업체 처분 의사결정(매각, 가업승계, 절충, 가족 합의안)을 AI가 세금 시뮬레이션, 노후 포트폴리오, 은퇴 자산 생존 확률, 가족 협상, 이상거래 보호까지 통합 지원한다.

### 1-2. 시스템 구성

사용자 입력(Streamlit UI)이 LangGraph 오케스트레이터로 전달되며, 처리 순서는 다음과 같다.

| 순서 | 노드 | 역할 | 분기 조건 |
|---|---|---|---|
| 1 | Supervisor | 질문 분석, 투입 에이전트 동적 선택 | 항상 실행 |
| 2 | Profiler | 프로필 로드, 필수 정보 확인 | 정보 누락 시 추가 질문을 반환하고 종료 |
| 3 | 병렬 실행 | BusinessValuation, TaxSuccession, PostExitWM, Negotiation 4개 에이전트 동시 실행 | 각 에이전트가 선택 여부를 판단해 자체 생략 |
| 4 | Synthesizer | 전문 에이전트 결과 종합, 시나리오 권고(A/B/C/D 중 1개) | 항상 실행 |
| 5 | SlowUIAdapter | 시니어 화면용 쉬운 표현 변환 | 시니어 모드일 때만 |
| 6 | ComplianceGuard | 금소법·세무 면책 검수 | 미통과 시 Synthesizer 재생성, 최대 3회 |
| 7 | FamilyBridge | 자녀 앱 리포트 발송 | 가족 공유 요청 시 |
| 8 | Booking | PB·세무사 상담 예약 | 예약 요청 시 |

대화 상태는 SQLite 체크포인터에 thread_id 단위로 영속 저장되어, 이어지는 질문에서도 이전 프로필과 시뮬레이션 결과가 유지된다.

병렬 실행 구간의 에이전트는 서로의 중간 결과를 참조할 수 없으므로, 타 에이전트 산출이 필요한 경우 공용 계산 모듈(tools)을 직접 호출하는 방식으로 일관성을 보장한다.

### 1-3. AI Agent 동작 원리 (이해 - 판단 - 행동 - 검증/개선)

본 시스템은 상황과 데이터를 이해하고, 필요한 정보를 바탕으로 판단하며, 적절한 행동을 자율적으로 선택하고, 결과를 검증·개선하는 Agent형 동작 구조를 갖는다.

| 단계 | 정의 | 본 시스템의 구현 |
|---|---|---|
| 이해 (인지) | 상황과 데이터를 이해 | 자연어 질문의 의도 분석(Supervisor), 고객 프로필과 삶의 조건 로드(Profiler), 거래내역·매출 이력 등 데이터 수집(FraudGuard, 조기경보), 음성 입력의 텍스트 변환(STT) |
| 판단 | 필요한 정보를 바탕으로 판단 도출 | 투입 에이전트 결정(Supervisor), 필수 정보 충족 여부 판단(Profiler), 세금·포트폴리오·생존 확률 계산에 근거한 시나리오 권고 — A/B/C/D 중 1개를 구조화 출력으로 강제(Synthesizer), 이상거래 위험도 분류(FraudGuard) |
| 행동 | 판단 결과에 따라 다음 행동을 선택 | 추가 질문(Profiler — 정보 누락 시), 근거 조회(세법 RAG 검색), 수정안 생성(검수 지적사항을 반영한 응답 재생성), 알림(FamilyBridge 가족 리포트, FraudGuard 이상거래 가족 통지), 요약(Synthesizer 종합 의견, SlowUIAdapter 시니어 변환, TTS 음성 브리핑), 예약(Booking — PB·세무사 상담) |
| 검증/개선 | 결과를 점검하고 필요 시 재시도 또는 종료 조건으로 품질 관리 | ComplianceGuard가 금소법·세무 면책 기준으로 모든 응답을 점검하고, 미통과 시 지적사항을 피드백하여 재생성(최대 3회 종료 조건, 초과 시 PB 연결로 전환). 수치 정확성은 규칙 기반 계산기와 단위 테스트 39건으로 상시 검증 |

## 2. AI Agent 상세 설계

### 2-1. Supervisor Agent (질문 분석·에이전트 라우팅)

| 항목 | 내용 |
|---|---|
| 역할 | 사용자 질문 분석 후 필요한 전문 에이전트 목록을 동적으로 결정 |
| 모델 | GPT-4o-mini (temperature 0) |
| 입력 | 사용자 자연어 질문 |
| 출력 | selected_agents 목록 |
| 안전장치 | 핵심 3개 에이전트가 모두 미선택되면 전부 기본 투입. 가족·협상 관련 라우팅은 LLM 판단과 별개로 키워드 기반 결정론 규칙을 우선 적용 |

선택 기준

| 질문 유형 | 투입 에이전트 |
|---|---|
| 종합 상담 (예: 어떻게 해야 할까) | BusinessValuation, TaxSuccession, PostExitWM |
| 세금·절세·증여 단독 | TaxSuccession |
| 자산운용·포트폴리오·노후 단독 | PostExitWM |
| 권리금·시세·가치평가 단독 | BusinessValuation |
| 가족 공유 요청 (딸·아들·자녀·공유·전달 등) | 해당 에이전트에 FamilyBridge 추가 |
| 협상 조건 언급 (협상·합의안·승계 비율 제안 등) | 해당 에이전트에 Negotiation 추가 |

### 2-2. Profiler Agent (사용자 프로필 로드·추가 질문)

| 항목 | 내용 |
|---|---|
| 역할 | DB에서 고객 프로필 로드, 핵심 정보 누락 시 1개 추가 질문 생성 |
| 모델 | GPT-4o-mini |
| 필수 확인 항목 | 월 순이익, 운영 기간, 권리금(예상 매도가) |
| 출력 | user_profile, clarification_needed(추가 질문 또는 빈 값) |
| UI 모드 결정 | 프로필의 시니어 설정에 따라 SlowUIAdapter 활성화 |

### 2-3. BusinessValuation Agent (사업체 가치평가)

| 항목 | 내용 |
|---|---|
| 역할 | 권리금·보증금·시설을 포함한 총 사업체 가치 산출 |
| 구성 | 규칙 기반 계산(수익환원법)과 GPT-4o-mini 설명 생성 |
| 계산 방식 | 권리금 = 월 순이익 x 업력 기반 배수 |
| 출력 항목 | 권리금 추정액, 보증금, 시설·집기, 합계, 고객 눈높이 설명문 |

업력별 권리금 배수

| 업력 | 등급 | 배수 |
|---|---|---|
| 10년 미만 | C (신생) | 월 순이익의 24개월분 |
| 10년 이상 20년 미만 | B (안정) | 월 순이익의 30개월분 |
| 20년 이상 | A (우량) | 월 순이익의 36개월분 |

### 2-4. TaxSuccession Agent (세금·승계 시뮬레이션)

| 항목 | 내용 |
|---|---|
| 역할 | 매각·가업승계·일반증여 3개 시나리오 세금 자동 계산 및 비교 |
| 구성 | 규칙 기반 세금 계산기, RAG(2025년 개정 세법, ChromaDB), GPT-4o 설명 생성 |

시나리오별 계산 기준 (국세청 공식 세율)

| 시나리오 | 세목 | 계산 기준 |
|---|---|---|
| 외부 매각 | 권리금 기타소득세 | 권리금의 40%를 과세표준에 산입(필요경비 60% 인정), 사업소득과 합산하여 종합소득세 누진세율(6~45%) 적용, 지방소득세 10% 가산 |
| 가업승계 과세특례 | 증여세 (조세특례제한법 제30조의6) | 10억원 공제 후 세율 10%(과세표준 60억원 초과분 20%), 한도 600억원, 사후관리 요건 안내 포함 |
| 일반 증여 | 증여세 | 성인 자녀 기본공제 5,000만원, 누진세율 10~50% |

계산용 세율표와 화면 표시용 세율표는 단일 모듈(tools/calculators.py)을 공통 참조하여 수치 불일치를 구조적으로 차단하며, 대표 수치는 단위 테스트로 고정되어 있다.

### 2-5. PostExitWM Agent (은퇴 후 자산운용·생존 확률 시뮬레이션)

| 항목 | 내용 |
|---|---|
| 역할 | 시나리오별 노후 월 수령액 포트폴리오 설계와 은퇴 자산 생존 확률 분석 |
| 구성 | 규칙 기반 포트폴리오 계산, 몬테카를로 시뮬레이션(tools/monte_carlo.py), GPT-4o 비교 설명 |
| 상품 기반 | JB자산운용·JB생명·전북은행·광주은행 실제 상품 카탈로그 |

시나리오 구성

| 시나리오 | 설명 | 포트폴리오 성향 |
|---|---|---|
| A. 완전 매각 | 사업체 전체 매각, 세후 현금 전액 운용 | 보수·균형 (상권 트렌드, 은퇴 시기 반영) |
| B. 완전 승계 | 자녀에게 전부 이전, 자문료로 소득 보완 | 성장·균형 |
| C. 절충안 | 권리금 50% 현금화, 나머지 승계 | 균형 |

포트폴리오 구성 (균형형 기준 4분산)

| 상품 유형 | 비율 | 상품 예시 |
|---|---|---|
| 월배당펀드 | 30% | JB자산운용 월배당 혼합채권형 펀드 |
| 즉시연금 | 28% | JB생명 즉시연금보험 (사업비 7% 차감 반영) |
| 정기예금 | 27% | 전북은행·광주은행 정기예금 |
| IRP (퇴직연금) | 15% | 전북은행 IRP |

- 모든 수령액은 세후 기준(이자·배당소득세 15.4%, 연금소득세 5.5% 차감)으로 산출한다.
- 주택연금(한국주택금융공사 공시, 종신 정액형)은 활용 선택 시 모든 시나리오에 가산한다.
- 0·5·10·15·20년 후 명목·실질 월 수령액과 잔여 원금의 장기 전망을 제공한다.

은퇴 자산 생존 확률 (몬테카를로 시뮬레이션)

| 항목 | 내용 |
|---|---|
| 모형 | 자산 수익률은 월 단위 정규분포, 긴급 의료비는 복합 포아송 과정(연평균 0.25회 발생, 1회 평균 3,000만원 지수분포)으로 모형화 |
| 시뮬레이션 | 1,000회 경로, 100세까지 월 단위, 결과 재현성을 위한 난수 시드 고정 |
| 통제 비교 | 모든 시나리오가 동일한 시장 국면과 의료비 쇼크를 겪도록 공통 난수를 적용하여 시나리오 간 순수 비교 보장 |
| 순인출 정의 | 목표 생활비에서 국민연금·주택연금·자문료 수입을 차감한 금액 (자문료는 10년 한정) |
| 수익률 가정 | A안 연 4.5%(변동성 6.0%), B안 3.0%(1.5%), C안 3.8%(4.0%) |
| 출력 | 연령별 생존 확률 곡선, 자산 고갈 연령 구간 분포, 100세 시점 생존 확률, 잔여 자산 중앙값과 하위 10% |

### 2-6. ComplianceGuard Agent (금소법·세무 면책 검수)

| 항목 | 내용 |
|---|---|
| 역할 | 최종 응답이 금소법·세무 면책 기준을 충족하는지 자동 검수 |
| 모델 | GPT-4o-mini (temperature 0) |
| 최대 재시도 | 3회 (미통과 시 PB·세무사 연결로 전환) |
| 적용 범위 | 본 분석 응답, 후속 질문 응답, 가족 합의안(D안) 설명문 |

검수 규칙

1. 세금 추정치에 약·예상·추정 등 불확실성 표현 명시
2. 세무사 상담 권장 문구 포함
3. 투자 추천 시 원금 손실 가능성과 위험등급 명시
4. 최종 결정은 고객 본인 또는 담당 PB·세무사가 해야 함을 명시
5. 특정 상품에 대한 단정적 최선 표현 금지

### 2-7. FamilyBridge Agent (가족 리포트 공유)

| 항목 | 내용 |
|---|---|
| 역할 | 시뮬레이션 결과를 자녀 JB 앱으로 발송 |
| 트리거 | 질문에 딸·아들·자녀·공유·전달·보내 등 키워드 포함 시 |
| 발송 내용 | 사업체 가치, 시나리오별 세금, 은퇴 후 월 수령액 요약 |

### 2-8. Negotiation Agent (가족 협상 AI — 핵심 차별화 기능)

| 항목 | 내용 |
|---|---|
| 역할 | 자녀의 협상 조건을 반영한 가족 합의안(D안) 산출 |
| 파이프라인 통합 | 본 분석 파이프라인의 4번째 병렬 노드로 동작. D안 결과는 Synthesizer 종합 권고와 ComplianceGuard 검수를 동일하게 거침 |
| 진입 경로 1 | 자녀 화면의 협상 폼 (승계 비율 슬라이더, 자문료율 슬라이더, 자녀의 한마디) |
| 진입 경로 2 | 부모의 대화 입력 (예: 딸이 70% 승계에 자문료 20%를 제안하면 어떻게 되나요) — 질문에서 승계율·자문료율을 자동 추출하며, 추출 실패 시 기본값 70%·20% 적용 |
| 안전장치 | 협상 키워드 또는 협상 조건 입력이 있을 때만 동작하는 결정론 게이트를 두어, 일반 질문에서 합의안이 생성되지 않도록 차단 |
| 모델 | GPT-4o (합의안 설명), GPT-4o-mini (조건 추출) |
| 출력 | 부모 월 수령 합계, 자녀 10년 누적 수익, 10년 후 권리금 추정, 합의안 설명문 |

D안 계산 절차

1. 부모 현금화 금액 = 권리금 x (1 - 승계 비율)
2. 현금화 금액에 권리금 기타소득세를 차감하고 개인 저축을 합산하여 운용자산 구성
3. 자녀 자문료 = 월 순이익 x 자문료율 (10년간 부모에게 지급)
4. 운용자산을 균형형 포트폴리오로 설계하고 월 수령액 산출
5. 가업 지속 가치(자녀 누적 수익, 10년 후 권리금)를 승계 비율로 안분

### 2-9. Booking Agent (PB·세무사 상담 예약)

| 항목 | 내용 |
|---|---|
| 역할 | 시뮬레이션 이후 PB 또는 세무사 상담 예약 자동 연결 |
| 트리거 | 질문에 예약·상담·방문·세무사 연결·PB 연결 등 포함 시 |
| 출력 | 예약 확인 메시지, 지점·담당자 정보, 준비 서류 목록 |

### 2-10. FraudGuard (이상거래 감지·소비흐름 분석·가족 알림)

| 항목 | 내용 |
|---|---|
| 역할 | 고령 고객 계좌의 이상거래를 감지하여 보호자(자녀)에게 알림하고, 소비흐름을 분석해 생활비 설계 근거를 제공 |
| 구성 | 규칙 기반 탐지 엔진 (재현성과 오탐 통제를 위해 LLM 미사용) |
| 분석 대상 | 최근 90일 개인 거래내역 (현재 데모 데이터) |
| 가족 알림 | 감지 시 자녀에게 거래 내역·위험 사유·지급정지 안내(전북은행 고객센터, 경찰청)를 포함한 알림 발송 |
| 역할 경계 | 거래의 실시간 차단은 은행 고유 FDS 영역. 본 기능은 고객 동의 기반 거래내역 조회를 활용한 보조 감시·가족 통지에 집중 |
| 데이터 연동 로드맵 | 1단계(대회): 합성 거래 데이터와 금융보안원 마이데이터 테스트베드 모의 연동. 2단계(파일럿): 전북은행 마이데이터 서비스 기반 거래내역 조회와 알림톡 발송. 3단계(내부 적용): 서비스의 행내 편입 시 JB은행 FDS 연계 |

탐지 규칙

| 규칙 | 기준 | 대응 사기 유형 |
|---|---|---|
| 고액 이탈 | 평소 지출 분포(평균 + 3 표준편차) 이탈 및 100만원 초과 | 편취 이체 |
| 심야 거래 | 00시~06시 거래 | 심야 원격제어 피해 |
| 분할 결제 | 같은 날 동일 가맹점 반복 결제 합계 200만원 초과 | 한도 회피형 상품권 결제 |
| 해외 거래 | 해외 가맹점 거래 | 해외 부정 결제 |

복수 규칙에 해당하면 위험도 높음, 단일 규칙은 주의로 분류한다. 소비흐름 분석은 이상거래를 제외한 월평균 생활 지출과 카테고리별 지출을 산출하여 목표 생활비 설정의 근거 데이터로 활용한다.

### 3-2. 대화 영속성

SQLite 체크포인터(jb_legacy_memory.db)에 thread_id 단위로 대화 상태 전체를 저장한다. 사용자가 이어서 질문해도 이전 프로필과 시뮬레이션 결과가 유지된다.

### 3-3. LLM 호출 관리

모든 LLM 호출은 단일 팩토리 모듈(agents/llm.py)을 경유한다. API 키 미설정 시 명확한 안내와 함께 실행을 중단하며, 환경변수로 모델을 교체할 수 있어 코드 수정 없이 신규 모델 검증·전환이 가능하다.

## 7. 핵심 시나리오 시연 흐름

시연 조건: 이사장(62세, 전주, 30년 한정식, 월 순이익 450만원), 자녀 승계 의향 예, 상권 보합, 은퇴 3년 이내, 목표 생활비 월 300만원, 주택연금 활용(월 66만 3,000원)

1. 질문 입력: 가게를 딸한테 넘겨야 할지 팔아야 할지 모르겠어요
2. 질문 분석: BusinessValuation, TaxSuccession, PostExitWM 투입 결정
3. 프로필 로딩: 이사장 프로필 확인 (월 순이익 450만원, 30년 운영)
4. 병렬 분석
   - 사업체 가치: 총 3억 1,200만원 (권리금 1억 6,200만원, 보증금 1억원, 시설 5,000만원)
   - 세금 비교: 매각 약 2,875만원, 가업승계 특례 0원(10억원 공제 적용), 일반 증여 약 4,664만원
   - 포트폴리오: A안 월 약 312만원, B안 월 약 288만원, C안 월 약 255만원 (세후, 주택연금 포함)
   - 가업 지속 가치(B안): 자녀 10년 누적 수익 약 4억 5,600만원, 10년 후 권리금 약 1억 7,900만원, 가족 총자산 증가 약 6억 3,500만원
   - 생존 확률: 시나리오별 곡선과 고갈 연령 분포 제시. 자문료 종료(10년) 이후 유동자산 고갈 위험이 정량 확인되어 주택연금 활용과 지출 조정의 효과를 함께 안내
5. 종합 의견: 시나리오 비교와 권고, 계산 근거(세율 구간·수식) 자동 첨부
6. 금소법 검수: 세무사 권장·원금손실 경고 등 자동 확인
7. 가족 공유: 이과장 화면으로 리포트 발송
8. 가족 협상: 이과장이 승계 70%, 자문료 20% 제안 — 또는 이사장이 대화로 "딸이 70% 승계에 자문료 20%를 제안하면?"이라고 질문
   - D안 산출: 부모 월 수령 약 317만원(주택연금 포함), 자녀 10년 누적 수익 약 3억 2,000만원
   - D안이 A·B·C와 동일 기준으로 비교되어 종합 권고에 반영됨
9. 거래 안심: 최근 90일 거래에서 심야 고액 이체 등 4건 감지, 이과장에게 알림 발송
10. 상담 예약: 전북은행 전주본점 WM센터 예약과 준비 서류 안내
