# JB Legacy — MVP 제안서

**팀명:** JB Legacy
**제출일:** 2026년 6월
**주제 분류:** 지정주제 1 — 개인 라이프케어 AI (LifeLong WM AI Agent 서비스 개발)

---

## 1. 문제 정의

### 1-1. 서비스 배경 및 주제 적합성

JB금융그룹의 핵심 영업 기반인 전북·전남·광주 지역에는 30년 내외 업력의 자영업자가 다수 존재한다. 이들은 60대 전후 은퇴 시점에 생애 최대 자산인 사업체의 처분 방향(매각, 가업승계, 절충)을 결정해야 하는 복합적 금융 과제에 직면한다.

본 서비스는 이 의사결정을 사업체 가치평가, 세금 시뮬레이션, 은퇴 후 현금흐름 설계, 가족 협상, 건강·사기 리스크 관리까지 통합 지원하는 자영업자 특화 LifeLong WM AI Agent이다. 자산 증식이 아닌 은퇴 이후의 현금흐름·소비·리스크 관리에 초점을 두고, 고령 고객의 디지털 장벽을 낮추는 적응형 UI를 제공한다는 점에서 지정주제 1의 문제의식과 직접 부합한다.

### 1-2. 대상 사용자 및 사용 상황

| 구분 | 설명 |
|------|------|
| 주 사용자 | 전북·광주 지역 50~65세 자영업자 (음식업·소매업 등 30년 내외 업력) |
| 보조 사용자 | 수도권 거주 자녀 (승계 협상 당사자, 보호자) |
| 사용 상황 | JB은행 PB 창구 방문 전 사전 시뮬레이션, 모바일 앱 자가 진단 |
| 대표 페르소나 (부) | 이사장, 62세, 전북 전주, 30년 한정식 운영, 총 사업체 가치 약 3억 1,200만원 |
| 대표 페르소나 (자녀) | 이과장, 32세, 서울 직장인, 이사장의 딸이자 승계 협상 상대방 |

### 1-3. 현행 업무와 서비스의 한계

고객(자영업자) 관점

- 사업체 가치(권리금·보증금·시설)를 스스로 산정하기 어렵고, 인터넷 정보는 업종·지역 편차가 커 신뢰하기 어렵다.
- 외부 매각 시 권리금 기타소득세, 가업승계 시 증여세 과세특례 등 세금 구조가 복잡하여, 세무사 상담 전까지 어떤 선택이 유리한지 알지 못한 채 의사결정이 이루어진다.
- 매각 후 노후 현금흐름을 수익률·세후 기준으로 비교하고, 의료비 등 돌발 지출 리스크까지 반영해 자산 고갈 가능성을 점검할 수 있는 도구가 없다.
- 부모와 자녀 간 승계 의사 조율 과정이 단절되어 있어 감정적 갈등이 발생한다.
- 고령 고객은 보이스피싱 등 금융사기에 노출될 가능성이 높으나, 가족이 이상 징후를 적시에 인지하기 어렵다.

JB금융그룹(기관) 관점

- PB 한 명이 가치평가·세무·자산운용을 모두 다루는 상담은 1건당 평균 2~3시간이 소요되어 대기 시간과 인력 부담이 크다.
- 자영업자 은퇴 자산(평균 1~5억원 규모)은 JB은행·JB자산운용 상품으로 연결될 수 있는 핵심 유치 기회임에도, 의사결정의 복잡성 때문에 타 대형사로 이탈하는 사례가 빈번하다.

### 1-4. 문제의 중요성 (JB금융그룹 관점)

- 전북·전남·광주 지역 자영업자는 약 24만 명(통계청, 2023)으로 JB금융의 주거래 고객층과 직접 겹친다.
- 60대 은퇴 자영업자의 사업체 평균 처분가는 1~5억원 수준이며, 이 자금이 정기예금·IRP·즉시연금으로 전환되면 JB은행·JB생명의 직접적 수익 기반이 된다.
- 가업승계 과세특례(조세특례제한법 제30조의6) 활용 시 자녀의 JB은행 사업자 대출 수요로 연결된다.
- 그룹의 AI 전환(AX) 전략 및 디지털 전환 방향과 일치한다.

## 2. 금융업무·사업 연계성 및 고객가치

### 2-1. JB금융그룹 사업 영역과의 연결

| 서비스 기능 | 연결 JB 계열사 | 연결 방식 |
|---|---|---|
| 사업체 가치평가 | 전북은행·광주은행 PB | 상담 사전 진단 자료 자동 생성 |
| 세금·승계 시뮬레이션 | 세무 제휴 네트워크 | 세무사 연계 예약 자동화 |
| 매각 후 포트폴리오 설계 | JB자산운용·JB생명·전북은행 | 월배당펀드·즉시연금·IRP·정기예금 실제 상품 기반 추천 |
| 은퇴 자산 생존 확률 분석 | 전북은행 PB | 의료비 쇼크 반영 몬테카를로 결과를 상담 자료로 제공 |
| 이상거래 감시·가족 알림 | 전북은행 FDS·JB 모바일 앱 | 고령 고객 계좌의 이상 징후를 보호자에게 통지 |
| 상담 예약 | 전북은행·광주은행 PB | AI가 예약 의뢰와 준비 서류 안내 자동 생성 |
| 가족 리포트 공유 | JB 모바일 앱 | 자녀 앱으로 시뮬레이션 결과 발송 |
| PB 상담용 PDF 리포트 | 전북은행·광주은행 PB | 분석 결과·계산 근거·차트를 표준 양식 문서로 자동 출력 |

### 2-2. 고객 여정 내 활용 접점

1. 인지 — 전북은행 지점 안내문, 모바일 앱 배너("내 가게, 얼마에 팔 수 있을까?")를 통한 유입
2. 자가 진단 — 사업체 정보 입력 후 가치평가·세금·포트폴리오·생존 확률 시뮬레이션을 즉시 제공
3. 가족 공유·협상 — 자녀 앱으로 리포트 발송, 자녀가 협상 조건을 입력하면 가족 합의안(D안)을 자동 생성. 부모가 대화 중 협상 조건을 말하면 동일한 합의안 분석이 본 분석 흐름 안에서 수행됨
4. 상시 보호 — 거래 안심 기능이 소비흐름을 분석하고 이상거래 감지 시 가족에게 알림
5. PB 상담 예약 — AI가 준비한 상담 자료(프로필, 시뮬레이션 결과, PDF 리포트)와 함께 예약 확정
6. 자산 전환 — JB 상품(즉시연금·IRP·월배당펀드·정기예금) 가입으로 연결

### 2-3. 기대효과

| 항목 | 내용 |
|---|---|
| 고객 가치 | PB 상담 대기 2~3주를 즉시 시뮬레이션으로 대체. 복잡한 세금 비교를 수 분 내 시각화 |
| 업무 효율 | PB 상담 준비 시간 단축. 고객이 시뮬레이션 결과와 PDF 리포트를 지참하고 방문 |
| 리스크 감소 | 금소법·세무 면책 검수 AI 자동 수행으로 불완전판매 위험 감소. 이상거래 조기 통지로 고령 고객 사기 피해 예방 |
| 수익 기회 | 은퇴 자산(1~5억원)의 JB 상품 전환율 향상, 자녀 사업자대출 신규 발굴 |

### 2-4. 규제·보안·내부통제 고려사항

- 금융소비자보호법: ComplianceGuard 에이전트가 모든 응답을 자동 검수한다. 세금 추정치의 불확실성 명시, 세무사 상담 권장 문구 삽입, 원금 손실 가능성 경고를 의무화하며, 가족 합의안(D안) 설명문도 동일한 검수를 거친다.
- 개인정보보호법: 현재 로컬 SQLite 저장. 향후 JB 내부 프라이빗 클라우드 전환 예정.
- 설명가능성: 세금·수익률·생존 확률 등 모든 수치는 규칙 기반 계산기가 산출하고, 계산식과 적용 세율 구간을 화면과 PDF 리포트에 그대로 노출한다. 계산용 세율표와 표시용 세율표를 단일 출처로 관리하여 불일치를 차단한다.
- 책임소재: 최종 결정은 고객 본인 또는 담당 PB·세무사가 해야 함을 모든 응답에 명시한다.

### 2-5. 그룹 AX·디지털 전환 취지와의 부합

- PB 업무 보조 AI 자동화라는 그룹 AX 전략의 현장 적용 사례이다.
- LangGraph 기반 멀티에이전트 아키텍처는 향후 대출심사·디지털 마케팅 등 다른 업무 도메인으로 확장 가능한 그룹 공통 플랫폼 기반이 된다.

## 3. 혁신성·확장성 및 운영 리스크 관리

### 3-1. 기존 서비스 대비 차별성

| 비교 항목 | 기존 PB 상담 방식 | JB Legacy AI |
|---|---|---|
| 가치평가 | PB 수기 계산, 수일 소요 | 수익환원법 알고리즘 즉시 산출 |
| 세금 비교 | 세무사 별도 상담 필요 | RAG 기반 세법 검색과 3개 시나리오 자동 계산 |
| 포트폴리오 | 경험 기반 추천 | JB 실제 상품 기반 세후 수익률 산출 |
| 장수 리스크 | 정성적 안내 | 의료비 쇼크를 반영한 몬테카를로 생존 확률(100세 기준)과 자산 고갈 연령 분포 제시 |
| 가족 협상 | 가족이 직접 갈등 조율 | 자녀 협상 조건 입력 또는 대화 중 언급만으로 합의안(D안) 자동 산출 |
| 금융사기 보호 | 사후 대응 | 이상거래 자동 감지 및 가족 알림 |
| 금소법 검수 | PB 개인 판단 의존 | AI 자동 검수와 자동 수정 재시도 |
| 응답 속도 | 상담 대기 2~3주 | 즉시 (24시간 365일) |

### 3-2. 새로운 고객 경험

- 가족 협상 AI: 자녀가 승계 조건(승계 비율, 자문료율)을 입력하거나 부모가 대화 중 조건을 언급하면, AI가 중립적 합의안(D안)을 본 분석 파이프라인 안에서 도출하고 다른 시나리오와 동일 기준으로 비교·권고한다.
- 은퇴 자산 생존 확률 분석: 긴급 의료비 지출을 복합 포아송 과정으로 모형화한 몬테카를로 시뮬레이션(1,000회 경로, 100세까지)으로 시나리오별 자산 생존 확률 곡선과 고갈 연령 분포를 제공한다. 주택연금 활용, 목표 생활비 조정 등 가정 변경 시 변경 전후 수치를 비교해 보여준다.
- 거래 안심: 최근 90일 소비흐름을 카테고리별로 분석하고, 평소 지출 패턴 대비 고액·심야·분할·해외 거래를 자동 감지하여 보호자(자녀)에게 알림을 발송한다. 거래 차단은 은행 고유 FDS의 영역이므로, 본 서비스는 고객 동의 기반 거래내역 조회를 활용한 보조 감시와 가족 통지 역할에 집중한다.
- 시니어 적응형 UI: 고령 고객 화면은 큰 글씨와 쉬운 표현으로 자동 변환되며, 음성 입력(STT)으로 말로 질문하고 음성 브리핑(TTS)으로 결과를 들을 수 있다.
- 시나리오 동시 비교: 매각·승계·절충·가족 합의안을 동시에 계산하여 월 수령액, 세금, 가족 자산 증가, 생존 확률을 한눈에 비교한다.

| AI 환각 | 규칙 기반 계산기로 세금·포트폴리오·생존 확률 수치를 선계산하고 LLM은 설명만 담당. 핵심 계산 함수는 단위 테스트 39건으로 기준 수치를 고정하여 회귀를 자동 검출 |

| 금소법 위반 | ComplianceGuard 에이전트가 통과 판정 전까지 최대 3회 자동 수정. 실패 시 PB 연결로 전환 |
| 모델 종속성 | LLM 호출을 단일 팩토리 모듈로 일원화하고 환경변수로 모델을 교체할 수 있어, 코드 수정 없이 신규 모델 검증·전환 가능 |
| 저작권·라이선스 | OpenAI API(상용), LangGraph(MIT), ChromaDB(Apache 2.0), Streamlit(Apache 2.0), fpdf2(LGPL 3.0) — 모두 상업적 이용 가능 |
| 세금 정보 정확성 | RAG 문서는 2025년 개정 세법 기준 수동 검수. 분기별 업데이트 정책 수립 예정 |
| 책임소재 | 모든 응답 말미에 참고용 시뮬레이션임을 고지하고 담당 PB·세무사 상담을 안내 |

## 4. 서비스 개요 요약

**서비스명:** JB Legacy — 자영업자 AI 은퇴 플래닝 어시스턴트

**한 줄 정의:**
전북·광주 지역 자영업자가 은퇴 시점에 사업체를 팔지, 자녀에게 넘길지, 절충할지를 AI가 세금·포트폴리오·생존 확률·가족 협상·사기 보호까지 한 번에 시뮬레이션해 주는 생애 관리형 서비스

**핵심 가치 제안:**

1. 세금 3개 시나리오(매각·가업승계 특례·일반증여) 즉시 비교
2. JB 실제 상품 기반 노후 월 수령액 포트폴리오와 100세 생존 확률 제시
3. 부모·자녀 협상 AI로 가족 갈등 없이 합의안(D안) 도출
4. 이상거래 감지와 가족 알림으로 고령 고객 금융사기 예방
5. 음성 입력·음성 브리핑·큰 글씨 화면의 시니어 적응형 UI
6. 금소법 자동 검수로 불완전판매 리스크 차단

**기술 스택:**
Python 3.14, LangGraph 1.2, LangChain 1.3, OpenAI GPT-4o·GPT-4o-mini·Whisper·TTS, ChromaDB(RAG), SQLite(체크포인터·mock 데이터), Streamlit 1.57, fpdf2(PDF 리포트)

**품질 검증 체계:**
세금 계산·몬테카를로·이상거래 탐지에 대한 단위 테스트 39건(기준 수치 고정)과 화면 자동화 테스트(Playwright) 9건을 갖추어 수치 회귀와 화면 오류를 자동 검출한다.

## 5. 추가 기능의 데이터 출처 고지 (데모 시뮬레이션)

아래 기능은 시뮬레이션(데모) 데이터로 구현되었으며, 화면에 데모 데이터 표시를 명기한다. 본선에서는 실제 API 연동으로 교체한다.

| 기능 | 구현 상태 | 본선 연동 대상 | 현재 데이터 위치 |
|---|---|---|---|
| 권리금 동적 평가와 엑시트 타이밍 | 데모 데이터 구현 | JB카드 마이데이터 API | data/mock_mydata.py |
| 청년 창업가 매칭과 JB 인수 대출 | 데모 데이터 구현 | JB Legacy 플랫폼 가입 청년 DB | data/db.py (SQLite) |
| 10년 자문료 자동이체와 생애주기 알림 | 수식 기반 구현 | JB은행 자동이체 API, 알림 서비스 | 계산식 기반 |
| 폐업 조기경보 대시보드 | 데모 데이터 구현 | JB카드 마이데이터 API | data/mock_mydata.py |
| 거래 안심 (이상거래 감지·소비흐름·가족 알림) | 데모 데이터 구현 | 단계적 연동: 마이데이터 테스트베드(모의), 전북은행 마이데이터 거래내역 조회(파일럿), 행내 FDS 연계(내부 적용 단계) | data/db.py (SQLite, 90일 거래 50건) |

### 본선 전환 시 교체 우선순위

| 우선순위 | 교체 대상 | 연동 대상 | 비고 |
|---|---|---|---|
| 1 | data/mock_mydata.py (매출 이력) | JB카드 마이데이터 API (OAuth2) | 조기경보·권리금 동적 평가의 핵심 데이터 |
| 2 | data/db.py transactions 테이블 | 전북은행 마이데이터 기반 거래내역 조회 (고객 동의) | 거래 안심 기능의 실데이터 전환. 실시간 차단은 은행 고유 FDS 영역이므로 본 서비스는 보조 감시와 가족 알림에 집중 |
| 3 | data/db.py youth_candidates 테이블 | JB Legacy 플랫폼 청년 창업가 DB | 서비스 운영 시 자연 축적 |
| 4 | agents/contract_manager.py | JB은행 자동이체·알림 서비스 API | 자문료 계약 자동화 |

### 고도화 시 추가 가능한 보조 지표

| 보조 지표 | 데이터 출처 | 적용 위치 |
|---|---|---|
| 반경 내 동일 업종 점포 수 | 소상공인 상권정보 API | 경쟁강도 보완 |
| 유동인구·추정 매출 | 통신사 유동인구 데이터 또는 공공데이터 | 입지 점수 보완 |

경쟁강도·입지 점수는 보조 지표로서 매출 이력 없이 단독으로는 조기경보 정확도가 낮으므로, 본선 연동 우선순위에서 후순위로 처리한다.
