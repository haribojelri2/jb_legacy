# JB Legacy — MVP 제안서

**팀명:** JB Legacy
**주제 분류:** 지정주제 1 — 개인 라이프케어 AI (LifeLong WM AI Agent 서비스 개발)
**제출일:** 2026년 6월

---

## 1. Summary

**서비스명:** JB Legacy — 자영업자 AI 은퇴 플래닝 어시스턴트

**한 줄 정의:** 전북·광주 지역 자영업자가 은퇴 시점에 사업체를 팔지, 자녀에게 넘길지, 절충할지를 AI 멀티에이전트가 세금·포트폴리오·생존 확률·가족 협상·금융사기 보호까지 한 번에 시뮬레이션해 주는 생애 관리형 서비스.

자산 증식이 아닌 은퇴 이후의 현금흐름·소비·리스크 관리에 초점을 두고, 고령 고객의 디지털 장벽을 낮추는 적응형 UI를 제공한다는 점에서 지정주제 1(LifeLong WM)의 문제의식과 직접 부합한다.

**핵심 가치 제안**

1. 세금 3개 시나리오(매각·가업승계 특례·일반증여) 즉시 비교
2. JB 실제 상품 기반 노후 월 수령액 포트폴리오와 100세 생존 확률 제시
3. 부모·자녀 협상 AI로 가족 갈등 없이 합의안(D안) 도출
4. 이상거래 감지와 가족 알림으로 고령 고객 금융사기 예방
5. 음성 입력·음성 브리핑·큰 글씨 화면의 시니어 적응형 UI
6. 금소법 자동 검수로 불완전판매 리스크 차단

**현재 상태:** 핵심 기능이 실제 동작하는 MVP(Streamlit) 구현 완료. 단위 테스트 33건, 화면 자동화 테스트 9건으로 검증.

## 2. 문제 정의 (Problem Definition)

### 2-1. 대상 사용자 및 사용 상황

| 구분 | 설명 |
|------|------|
| 주 사용자 | 전북·광주 지역 50~65세 자영업자 (음식업·소매업 등 30년 내외 업력) |
| 보조 사용자 | 수도권 거주 자녀 (승계 협상 당사자이자 보호자) |
| 사용 상황 | JB은행 PB 창구 방문 전 사전 시뮬레이션, 모바일 앱 자가 진단 |
| 대표 페르소나 (부) | 이사장, 62세, 전북 전주, 30년 한정식 운영, 총 사업체 가치 약 3억 1,200만원 |
| 대표 페르소나 (자녀) | 이과장, 32세, 서울 직장인, 이사장의 딸이자 승계 협상 상대방 |

### 2-2. 현행 업무와 서비스의 Pain Point

고객(자영업자) 관점

- 사업체 가치(권리금·보증금·시설)를 스스로 산정하기 어렵고, 인터넷 정보는 업종·지역 편차가 커 신뢰하기 어렵다.
- 외부 매각 시 권리금 기타소득세, 가업승계 시 증여세 과세특례 등 세금 구조가 복잡하여, 세무사 상담 전까지 어떤 선택이 유리한지 알지 못한 채 의사결정이 이루어진다.
- 매각 후 노후 현금흐름을 수익률·세후 기준으로 비교하고, 의료비 등 돌발 지출 리스크까지 반영해 자산 고갈 가능성을 점검할 도구가 없다.
- 부모와 자녀 간 승계 의사 조율 과정이 단절되어 감정적 갈등이 발생한다.
- 고령 고객은 보이스피싱 등 금융사기에 노출될 가능성이 높으나, 가족이 이상 징후를 적시에 인지하기 어렵다.

JB금융그룹(기관) 관점

- PB 한 명이 가치평가·세무·자산운용을 모두 다루는 상담은 1건당 평균 2~3시간이 소요되어 대기 시간과 인력 부담이 크다.
- 자영업자 은퇴 자산(평균 1~5억원)은 JB은행·JB자산운용 상품으로 연결될 수 있는 핵심 유치 기회임에도, 의사결정의 복잡성 때문에 타 대형사로 이탈하는 사례가 빈번하다.

### 2-3. 문제의 중요성 (JB금융그룹 관점)

- 전북·전남·광주 지역 자영업자는 약 24만 명(통계청, 2023)으로 JB금융의 주거래 고객층과 직접 겹친다.
- 60대 은퇴 자영업자의 사업체 평균 처분가 1~5억원이 정기예금·IRP·즉시연금형 예금으로 전환되면 전북은행·광주은행의 직접적 수익 기반이 된다.
- 가업승계 과세특례(조세특례제한법 제30조의6) 활용 시 자녀의 JB은행 사업자 대출 수요로 연결된다.
- 그룹의 AI 전환(AX) 전략 및 디지털 전환 방향과 일치하며, 실제 서비스 과제로 전환 가능한 수준이다.

## 3. 제안 솔루션 개요 (Solution Overview)

### 3-1. 4가지 의사결정 시나리오

본 서비스는 사업체 처분을 단순 "현금 많고 적음"이 아니라 부모 안정·가족 자산 지속·균형이라는 3축으로 분석하고, 자녀가 협상 조건을 제안하면 가족 합의안까지 같은 기준으로 비교한다.

| 시나리오 | 최적화 대상 | 핵심 |
|---|---|---|
| A. 완전 매각 | 부모 노후 안정 | 권리금 현금화 후 운용 포트폴리오 |
| B. 완전 승계 | 가족 자산 지속 | 자녀 가업 이전, 자문료로 부모 소득 보완 |
| C. 절충안 | 균형 | 권리금 50% 현금화 + 50% 승계 (JB우리캐피탈 양수도 대출 연계) |
| D. 가족 합의안 | 협상 | 자녀가 제안한 승계 비율·자문료 조건 기반 |

### 3-2. AI Agent 동작 구조 (판단 - 행동 - 검증/개선)

LangGraph 기반 멀티에이전트가 상황과 데이터를 이해하고, 판단하며, 적절한 행동을 자율 선택하고, 결과를 검증·개선하는 Agent형 구조로 동작한다.

| 단계 | 본 시스템의 구현 |
|---|---|
| 판단 | 질문 의도 분석으로 투입 에이전트 결정(Supervisor), 필수 정보 충족 여부 판단(Profiler), 계산 근거에 기반한 시나리오 권고를 A/B/C/D 중 1개로 강제(Synthesizer), 이상거래 위험도 분류(FraudGuard) |
| 행동 | 추가 질문, 근거 조회(세법 RAG), 수정안 생성(검수 반영 재생성), 알림(가족 리포트·이상거래 통지), 요약(종합 의견·시니어 변환·음성 브리핑), 예약(PB·세무사 상담) |
| 검증/개선 | 이중 검증 루프 — (1) ComplianceGuard가 금소법 기준으로 모든 응답을 점검, 미통과 시 재생성(최대 3회 종료 조건, 초과 시 PB 연결). (2) 적대 검증(공격자·방어자·심판 3-LLM 토론 채점)이 응답을 심층 검증, '재생성필요' 판정 시 개선 지시를 주입해 자동 재생성(최대 1회). 생존 확률이 목표(85%) 미달이면 지속가능 생활비를 역산해 개선안을 자동 제시. 수치 정확성은 규칙 기반 계산기와 단위 테스트로 상시 검증 |

### 3-3. 환각 차단 설계 (정확성 우선)

금융 도메인은 자율성보다 정확성·규제 준수가 우선이므로, 모든 수치(세금·포트폴리오·생존 확률·이상거래)는 규칙 기반 엔진과 결정론 시뮬레이션이 계산하고 LLM은 설명만 담당한다. 계산 근거(세율 구간·수식)는 LLM이 아니라 코드가 직접 조립해 응답에 삽입하여 수치 환각을 원천 차단한다.

## 4. 주요 기능 정의 (Key Features)

| 기능 | 내용 | 기존 PB 상담 대비 |
|---|---|---|
| 사업체 가치평가 | 수익환원법(월 순이익 x 업력 배수)으로 권리금·총 가치 즉시 산출 | PB 수기 계산 수일 → 즉시 |
| 세금 3시나리오 비교 | 매각·가업승계 특례·일반증여 세금을 RAG 세법 검색과 함께 자동 계산 | 세무사 별도 상담 → 자동 |
| 노후 포트폴리오 | 전북은행·광주은행 실제 상품(JB 리치100 즉시연금형·KJ 정기예금·IRP) 기반 세후 월 수령액 설계, 목표 생활비에 맞춘 동적 배분 | 경험 기반 → 실제 상품·동적 배분 |
| 은퇴 자산 생존 확률 | 의료비 쇼크를 반영한 몬테카를로로 100세 생존 확률 곡선과 고갈 연령 분포 제시, 목표 생존확률(85%) 미달 시 지속가능 생활비 자동 역산 | 정성적 안내 → 정량 분석·개선안 제시 |
| 가족 협상 AI (D안) | 자녀 협상 조건 입력 또는 대화 중 언급만으로 중립적 합의안을 본 분석 흐름 안에서 산출 | 가족이 직접 갈등 조율 → AI 조율 |
| 거래 안심 | 소비흐름 분석과 이상거래(고액·심야·분할·해외) 자동 감지, 가족 알림 | 사후 대응 → 사전 감지·통지 |
| 시니어 적응형 UI | 큰 글씨·쉬운 표현 자동 변환, 음성 입력(STT)·음성 브리핑(TTS) | 디지털 장벽 → 음성 대화 |
| 금소법 자동 검수 | 모든 응답을 금소법 5개 규칙으로 검수하고 미통과 시 자동 수정 | PB 개인 판단 → AI 자동 검수 |
| AI 적대 검증 (GAN) | 공격자·방어자·심판 3-LLM 토론으로 응답을 5개 기준(금융정확성·규정준수·논리일관성·수치근거·편향성) 채점, 미달 시 자동 개선 재생성 | 사람 사후 검토 → AI 상호 검증 |

가족 협상 AI와 의료비 쇼크 생존 확률 분석은 기존 금융 서비스에 전례가 드문 차별화 기능이다.

## 5. 데이터 및 기술 활용 (Data Tech Plan)

### 5-1. 기술 구성

| 구분 | 기술 | 용도 |
|---|---|---|
| AI 오케스트레이션 | LangGraph (멀티에이전트, 조건부 라우팅, 상태 체크포인터) | 판단-행동-검증 흐름 제어 |
| LLM | Anthropic Claude claude-opus-4-8 (분석·라우팅·검수·적대검증) — 팩토리 라우팅으로 GPT 계열 즉시 전환 가능 | 판단·설명 생성 |
| RAG | ChromaDB + OpenAI text-embedding-3-small | 2025년 개정 세법 근거 검색 |
| Rule Engine | tools/calculators.py, tools/monte_carlo.py | 세금·포트폴리오·생존 확률 결정론 계산 |
| 음성 | OpenAI Whisper(STT)·TTS | 시니어 음성 입력·브리핑 |
| 데이터 | SQLite(체크포인터·거래내역·청년 후보), JB 상품 카탈로그 | 상태 영속·mock 데이터 |
| UI / 문서 | Streamlit, fpdf2(PB 상담용 PDF 리포트) | 화면·산출물 |

LLM 호출은 단일 팩토리 모듈로 일원화하여 환경변수로 모델을 교체할 수 있다(Claude·GPT 이중화 — 장애 시 즉시 롤백). 모든 오픈소스·상용 API는 상업적 이용이 가능하다(LangGraph·LangChain MIT, ChromaDB·Streamlit Apache 2.0, fpdf2 LGPL 3.0, Anthropic·OpenAI 상용).

### 5-2. 데이터 출처와 본선 연동 계획

핵심 분석(세금·포트폴리오·생존 확률)은 공식 세율·상품·통계 기준의 실데이터 계산이다. 아래 보조 기능은 데모 시뮬레이션 데이터로 구현했으며 화면에 데모 데이터임을 명기하고, 본선에서 실제 연동으로 교체한다.

| 기능 | 본선 연동 대상 | 현재 데이터 |
|---|---|---|
| 폐업 조기경보 · 권리금 동적 평가 | JB카드 마이데이터 API (OAuth2) | data/mock_mydata.py (24개월 매출) |
| 거래 안심 (이상거래·소비흐름) | 마이데이터 테스트베드(대회) → 전북은행 마이데이터 거래내역 조회(파일럿) → 행내 FDS 연계(내부 적용) | data/db.py (90일 거래 50건) |
| 청년 창업가 매칭 | JB Legacy 플랫폼 가입 청년 DB | data/db.py (SQLite) |
| 자문료 자동이체 | JB은행 자동이체·알림 서비스 API | 계산식 기반 |

거래의 실시간 차단은 은행 고유 FDS의 영역이므로, 본 서비스는 고객 동의 기반 거래내역 조회를 활용한 보조 감시와 가족 통지 역할에 집중한다.

### 5-3. 규제·보안·내부통제

- 금융소비자보호법: ComplianceGuard가 모든 응답(가족 합의안 포함)을 자동 검수하여 불확실성 명시·세무사 상담 권장·원금 손실 경고를 의무화한다.
- 설명가능성: 모든 수치는 규칙 기반 계산 결과이며 계산식과 세율 구간을 화면·PDF에 그대로 노출한다(계산용·표시용 세율표 단일 출처 관리).
- 개인정보: 사용자 식별자는 내부 ID만 사용하고 이름·연락처를 외부 API로 전송하지 않는다. 향후 JB 프라이빗 클라우드 전환.
- 책임소재: 최종 결정은 고객 본인 또는 담당 PB·세무사가 함을 모든 응답에 명시한다.

## 6. 사용자 시나리오 / 유즈케이스 (User Scenario)

### 6-1. 고객 여정

1. 인지 — 전북은행 지점 안내문·모바일 앱 배너("내 가게, 얼마에 팔 수 있을까?")로 유입
2. 자가 진단 — 사업체 정보 입력 후 가치평가·세금·포트폴리오·생존 확률 시뮬레이션 즉시 제공
3. 가족 공유·협상 — 자녀 앱으로 리포트 발송, 자녀가 협상 조건을 입력하거나 부모가 대화 중 언급하면 합의안(D안) 자동 생성
4. 상시 보호 — 거래 안심이 소비흐름을 분석하고 이상거래 감지 시 가족에게 알림
5. PB 상담 예약 — AI가 준비한 자료(프로필·시뮬레이션·PDF 리포트)와 함께 예약 확정
6. 자산 전환 — 전북은행·광주은행 실제 상품(JB 리치100 즉시연금형·KJ 정기예금·IRP) 가입으로 연결

### 6-2. 대표 시연 (이사장)

조건: 자녀 승계 의향 예, 상권 보합, 은퇴 3년 이내, 목표 생활비 월 300만원, 주택연금 활용(월 66만 3,000원)

| 단계 | 결과 |
|---|---|
| 질문: 가게를 딸한테 넘길지 팔지 모르겠어요 | Supervisor가 가치평가·세금·자산운용 에이전트 투입 |
| 사업체 가치 | 총 3억 1,200만원 (권리금 1억 6,200만 + 보증금 1억 + 시설 5,000만) |
| 세금 비교 | 매각 약 2,875만원 / 가업승계 특례 0원(10억 공제) / 일반 증여 약 4,664만원 |
| 노후 월수령(세후, 주택연금 포함) | A 약 288만 / B 약 289만 / C 약 282만 (전북은행·광주은행 실제 예금/연금형, 목표 생활비 기반 동적 배분) |
| 가업 지속 가치(B안, 보합) | 자녀 10년 누적 4억 5,600만 + 10년 후 권리금 1억 7,900만 = 가족 총자산 6억 3,500만 |
| 생존 확률 | 시나리오별 곡선·고갈 연령 분포. 예금 위주 보수 운용이라 100세까지 생존확률은 0%대 → 시스템이 목표 85% 달성 지속가능 생활비를 자동 역산(A안 월 약 122만원 등)해 주택연금 확대·지출 조정의 근거를 정량 제시 |
| 가족 협상 | "딸이 70% 승계에 자문료 20% 제안" → D안 부모 월 약 292만, 자녀 10년 누적 약 3억 2,000만 |
| 거래 안심 | 최근 90일 중 심야 고액 이체 등 4건 감지, 이과장에게 알림 발송 |
| 상담 예약 | 전북은행 전주본점 WM센터 예약과 준비 서류 안내 |

전 과정에서 금소법 검수와 PB·세무사 상담 권장이 자동 적용된다.
