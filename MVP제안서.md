# JB Legacy — MVP 제안서

**팀명:** JB Legacy  
**제출일:** 2026년 5월  
**주제 분류:** 자유주제 — 자산관리 × 준법지원 × 미래사업

## 1. 문제 정의

### 1-1. 서비스 배경 및 주제 적합성

JB금융그룹의 핵심 영업 기반인 전북·전남·광주 지역에는 30년 이상 업력의 자영업자가 다수 존재한다. 이들은 60대 전후의 은퇴 시점에 접어들면서 생애 최대 자산인 사업체의 처분 방향(매각 vs 승계)을 결정해야 하는 복합적 금융 과제를 안고 있다. 본 서비스는 이 문제를 AI Agent로 해결하는 **자영업자 특화 은퇴 플래닝 플랫폼**이다.

### 1-2. 대상 사용자 및 사용 상황

| 구분 | 설명 |
|------|------|
| **주 사용자** | 전북·광주 지역 50~65세 자영업자 (음식업·소매업 등 30년 내외 업력) |
| **보조 사용자** | 수도권 거주 자녀 (승계 협상 당사자) |
| **사용 상황** | JB은행 PB 창구 방문 전 사전 시뮬레이션, 또는 모바일 앱에서 자가 진단 |
| **대표 페르소나 (부)** | 이사장 (62세, 전북 전주, 30년 한정식 운영, 총 사업체 가치 약 3.1억원) |
| **대표 페르소나 (녀)** | 이과장 (32세, 서울 직장인, 이사장의 딸 / 승계 협상 상대방) |

### 1-3. 현행 업무·서비스의 Pain Point

**고객(자영업자) 관점**

- 사업체 가치(권리금·보증금·시설)를 스스로 계산하기 어렵고, 인터넷 정보는 업종·지역 편차가 커 신뢰하기 힘들다.
- 외부 매각 시 권리금 기타소득세, 가업승계 시 증여세 과세특례 등 세금 구조가 복잡하다. 세무사 상담 전까지 어떤 선택이 유리한지 전혀 모르는 상태에서 의사결정이 이뤄진다.
- 매각 후 노후 현금흐름(연금·예금·펀드 조합)을 수익률·세후 기준으로 비교할 수 있는 도구가 없다.
- 부모와 자녀 간 승계 의사 조율 과정이 단절되어 있어 감정 갈등이 발생한다.

**JB금융그룹(기관) 관점**

- PB 한 명이 가치평가·세무·자산운용을 모두 소화하려면 상담 1건당 평균 2~3시간이 소요된다. 대기 시간과 인력 부담이 크다.
- 자영업자 은퇴 자산(평균 2~5억원 규모)은 JB은행·JB자산운용 상품으로 연결될 수 있는 핵심 유치 기회임에도, 복잡성 때문에 타 대형사로 이탈하는 경우가 빈번하다.

### 1-4. 문제의 중요성 (JB금융그룹 관점)

- 전북·전남·광주 지역 자영업자 수: 약 24만 명 (통계청 2023). JB금융의 주거래 고객층과 직접 겹친다.
- 60대 은퇴 자영업자의 사업체 평균 처분가는 1~5억원 수준으로, 이 자금이 정기예금·IRP·즉시연금으로 전환되면 JB은행·JB생명의 직접 수익원이 된다.
- 가업승계 과세특례(조세특례제한법 제30조의6) 활용 시 자녀의 JB은행 사업자 대출 수요도 연결된다.
- 그룹 AX 전략(AI 기반 업무 자동화) 및 디지털 전환 방향과 완전히 일치한다.

### 1-5. 서비스 과제로의 전환 가능성

아래 3단계로 실제 서비스화가 가능하다.

| 단계 | 내용 | 시점 |
|------|------|------|
| **PoC (현재)** | Streamlit 데모 — PB 상담 전 고객 자가 시뮬레이션 도구 | 현재 |
| **파일럿** | 전북은행 PB 창구 5개 지점 태블릿 비치, 상담 보조 도구 활용 | +3개월 |
| **정식 서비스** | JB금융 모바일 앱 내 "내 사업체 은퇴 플래닝" 메뉴 통합 | +12개월 |

## 2. 금융업무·사업 연계성 및 고객가치

### 2-1. JB금융그룹 사업 영역과의 연결

| 서비스 기능 | 연결 JB 계열사 | 연결 방식 |
|---|---|---|
| 사업체 가치평가 | 전북은행·광주은행 PB | 상담 사전 진단 자료 자동 생성 |
| 세금·승계 시뮬레이션 | 세무 제휴 네트워크 | 세무사 연계 예약 자동화 |
| 매각 후 포트폴리오 설계 | JB자산운용·JB생명·전북은행 | 월배당펀드·즉시연금·IRP·정기예금 실제 상품 기반 추천 |
| 상담 예약 | 전북은행·광주은행 PB | AI가 예약 의뢰 자동 생성 |
| 가족 리포트 공유 | JB 모바일 앱 | 자녀 앱으로 시뮬레이션 결과 Push 발송 |

### 2-2. 고객 여정(Customer Journey) 내 활용 접점

```
① 인지
   └─ 전북은행 지점 안내문 / 모바일 앱 배너 "내 가게, 얼마에 팔 수 있을까?"

② 자가 진단 (JB Legacy AI)
   └─ 사업체 정보 입력 → 가치평가·세금·포트폴리오 시뮬레이션 즉시 제공

③ 가족 공유
   └─ 자녀 앱으로 리포트 발송 → 자녀가 협상 조건 입력 → D안(합의안) 자동 생성

④ PB 상담 예약 (AI 연결)
   └─ AI가 준비한 상담 자료(프로필·시뮬레이션 결과)와 함께 예약 확정

⑤ 자산 전환
   └─ JB 상품(즉시연금·IRP·월배당펀드·정기예금) 가입으로 연결
```

### 2-3. 기대효과

| 항목 | 내용 |
|---|---|
| **고객 가치** | 평균 PB 상담 대기 2~3주 → 즉시 시뮬레이션 제공. 복잡한 세금 비교를 3분 내 시각화 |
| **업무 효율** | PB 상담 준비 시간 70% 단축. 고객이 시뮬레이션 결과를 가지고 방문 |
| **리스크 감소** | 금소법·세무 면책 검수 AI 자동 수행. 불완전판매 위험 감소 |
| **수익 기회** | 은퇴 자산(1~5억원) JB 상품 전환율 향상. 자녀 사업자대출 신규 발굴 |

### 2-4. 규제·보안·내부통제 고려사항

- **금융소비자보호법(금소법):** ComplianceGuard 에이전트가 모든 응답을 자동 검수. 세금 추정치에 불확실성 명시, '세무사 상담 권장' 문구 강제 삽입, 원금 손실 가능성 경고 의무화.
- **개인정보보호법:** 현재 로컬 SQLite 저장. 향후 JB 내부 프라이빗 클라우드 전환 예정.
- **설명가능성(XAI):** AI 추천 근거(수익률·세율·계산식)를 모두 UI에 노출. 블랙박스 추천 금지.
- **책임소재 명시:** "최종 결정은 고객 본인 또는 담당 PB·세무사가 해야 함"을 모든 응답에 명시.

### 2-5. 그룹 AX·디지털 전환 취지와의 부합

- JB금융그룹 AX(AI Transformation) 전략의 현장 적용 사례: PB 업무 보조 AI 자동화
- LangGraph 기반 멀티에이전트 아키텍처는 향후 대출심사·디지털 마케팅 등 다른 업무 도메인으로 확장 가능한 그룹 공통 플랫폼 기반이 된다.

## 3. 혁신성·확장성 및 운영 리스크 관리

### 3-1. 기존 서비스 대비 차별성

| 비교 항목 | 기존 PB 상담 방식 | JB Legacy AI |
|---|---|---|
| 가치평가 | PB가 수기 계산, 수일 소요 | 수익환원법 알고리즘 즉시 산출 |
| 세금 비교 | 세무사 별도 상담 필요 | RAG 기반 세법 검색 + 3시나리오 자동 계산 |
| 포트폴리오 | 경험 기반 추천 | JB 실제 상품 기반 세후 수익률 최적화 |
| 가족 협상 | 가족이 직접 갈등 조율 | 자녀 협상 조건 입력 → 합의안(D안) 자동 산출 |
| 금소법 검수 | PB 개인 판단 의존 | AI 자동 검수 + 3회 자동 수정 재시도 |
| 응답 속도 | 상담 대기 2~3주 | 즉시 (24시간 365일) |

### 3-2. 새로운 고객 경험

- **가족 협상 AI:** 부모·자녀가 각자 디바이스에서 승계 조건(승계 비율, 자문료율)을 입력하면 AI가 중립적 합의안을 도출한다. 금융 업계 전례 없는 기능.
- **느린 UI 모드:** 디지털 취약 계층(노년 고객)을 위해 텍스트가 천천히 타이핑 효과로 출력되고, 모든 수치는 한국어 설명과 함께 제공된다.
- **3시나리오 동시 비교:** 매각·승계·절충안을 동시에 계산하여 월 수령액·세금·가족 자산 증가를 한눈에 비교.

### 3-3. 확장 가능성

**계열사 확장**

| 계열사 | 확장 방향 |
|---|---|
| JB우리캐피탈 | 은퇴 전 사업 부채 최적화 기능 추가 |
| JB생명 | 즉시연금·종신보험 설계 자동화 |
| 광주은행 | 광주·전남 자영업자 동일 서비스 확장 |

**업무·고객군 확장**
- 농업인 은퇴 플래닝 (농지 매각 vs 임대 시뮬레이션)
- 소규모 법인 대표이사 은퇴 (주식 매각 vs 법인 청산)
- 30~40대: "10년 후 은퇴 시뮬레이션" 미래 플래닝

### 3-4. 발전 경로 (PoC → 서비스화)

```
[현재] PoC — Streamlit 데모, 로컬 실행
   ↓ +3개월
[파일럿] 전북은행 PB 창구 5개 지점, 태블릿 비치, 실사용 피드백 수집
   ↓ +6개월
[내부 적용] JB 내부망 배포, PB 보조 도구로 정식 채택
   ↓ +12개월
[고객 서비스화] JB 모바일 앱 "은퇴 플래닝" 메뉴 통합, 대고객 서비스 오픈
```

### 3-5. 운영 리스크 대응

| 리스크 유형 | 대응 방안 |
|---|---|
| **AI 환각 (Hallucination)** | Rule Engine 기반 세금 계산기로 수치 선계산 후 LLM은 설명만 담당. 수치 오류 원천 차단 |
| **개인정보 침해** | 사용자 식별자는 내부 ID만 사용. 이름·연락처 외부 API 전송 없음. 향후 프라이빗 클라우드 전환 |
| **금소법 위반** | ComplianceGuard 에이전트가 PASS 판정 전까지 최대 3회 자동 수정. 실패 시 PB 연결 |
| **저작권·라이선스** | OpenAI API (상용), LangGraph (MIT), ChromaDB (Apache 2.0), Streamlit (Apache 2.0) — 모두 상업적 이용 허가 |
| **세금 정보 정확성** | RAG 문서는 2025년 개정 세법 기준 수동 검수. 분기별 업데이트 정책 수립 예정 |
| **책임소재** | 모든 응답 말미에 "본 내용은 참고용 시뮬레이션이며, 최종 결정은 담당 PB·세무사와 상의하시기 바랍니다" 고지 |

## 4. 서비스 개요 요약

**서비스명:** JB Legacy — 자영업자 AI 은퇴 플래닝 어시스턴트

**한 줄 정의:**  
전북·광주 지역 자영업자가 은퇴 시점에 사업체를 팔지, 자녀에게 넘길지, 둘을 절충할지를 AI가 세금·포트폴리오·가족 협상까지 한 번에 시뮬레이션해주는 서비스

**핵심 가치 제안:**
1. 세금 3시나리오(매각·가업승계·일반증여) 즉시 비교
2. JB 실제 상품 기반 노후 월 수령액 최적 포트폴리오 제시
3. 부모·자녀 협상 AI로 가족 갈등 없이 합의안 도출
4. 금소법 자동 검수로 불완전판매 리스크 제거

**기술 스택:**  
Python 3.14 / LangGraph 0.2 / LangChain / OpenAI GPT-4o·GPT-4o-mini / ChromaDB (RAG) / SQLite (체크포인터) / Streamlit

### MOCK 데이터 구조 요약

**`data/mock_mydata.py`** (Feature 1·5 공용)

- 지역·업종별 청년 창업가 후보 6명 프로필 (SQLite)
- JB 대출 상품 정보 (`jb_products.py` → `jb_products` 테이블)
- `fetch_youth_candidates()`: 업종 유사도·지역 매칭 점수 기반 상위 N명 반환

### 본선 전환 시 교체 대상

```
MOCK 제거 대상 → 실 API 연동 대상
────────────────────────────────────────────────────────
[1순위] data/mock_mydata.py (매출 이력)  → JB카드 마이데이터 API (OAuth2)
        └ Feature 5·1의 핵심. 연동만 되면 조기경보·권리금 평가 모두 실데이터로 동작
[2순위] data/db.py (youth_candidates)   → JB Legacy 플랫폼 가입 청년 창업가 DB (서비스 자체 운영)
[3순위] agents/contract_manager.py     → JB 은행 자동이체·알림 서비스 API
```

### 고도화 시 추가 가능한 보조 지표

매출 이력(JB카드 마이데이터) 연동 이후 정확도 향상을 위해 아래 데이터를 보조 지표로 추가할 수 있다.

| 보조 지표 | 데이터 출처 | 적용 위치 |
|---|---|---|
| 반경 내 동일 업종 점포 수 | 소상공인 상권정보 API `/storeListInRadius` | 경쟁강도 보완 |
| 유동인구·추정 매출 | 통신사 유동인구 데이터 (유료) 또는 서울시 공공데이터 | 입지 점수 보완 |

> 경쟁강도·입지 점수는 보조 지표로, 매출 이력 없이 단독으로는 조기경보 정확도가 낮다. 본선 연동 우선순위에서 후순위로 처리한다.
