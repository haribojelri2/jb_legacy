# 준법·리스크 검토 하네스

금융 서비스는 자율성보다 정확성·규제 준수가 우선이다. 응답이 사용자에게 도달하는 모든 경로에서 규제 리스크를 점검한다.

## 점검 항목

1. **면책 고지(금소법) 커버리지**
   - 모든 사용자 응답 경로에 `[주의사항]` 면책이 붙는가:
     - synthesizer 다중 에이전트 경로 (`agents/synthesizer.py` disclaimer 조립)
     - synthesizer 단일 에이전트 경로
     - slow_ui 시니어 변환 (`agents/slow_ui_adapter.py` — 면책 원문 보존 확인)
     - UI 후속답변 경로 (`ui/app.py`)
   - `agents/compliance.py`의 5개 규칙(불확실성 명시·세무사 권장·원금손실·책임소재·단정 금지) 검수가 실제로 작동하는가

2. **수치 정합성(환각 차단)**
   - 금액이 LLM 생성이 아니라 규칙 기반 계산기 결과인가 (`tools/calculators.py`, `[계산 근거]` 코드 조립)
   - `agents/compliance.py`의 `verify_numbers()`가 계산-응답 금액을 결정론 대조하는가
   - LLM 프롬프트에 "마크다운 금지" 지시 + `agents/textutil.py` 새니타이저 이중 방어

3. **개인정보(PII)**
   - 사용자 이름·연락처가 외부 LLM 프롬프트로 전송되는가 (내부 ID만 사용 원칙 확인)
   - 가족 알림(`family_bridge`, fraud_guard `build_family_alert`)에 민감정보 노출 여부

4. **책임소재**
   - "최종 결정은 고객 본인/PB·세무사" 문구가 명시되는가
   - compliance 재시도 3회 초과 시 PB 연결 폴백이 실제 동작하는가 (`graph.py` _route_compliance)

5. **세법 도메인 정확성 (알려진 리스크)**
   - 조특법 30조의6 가업승계 특례를 개인사업자에 적용 중 — 실제 특례는 중소기업 '주식 등' 증여 대상. 프롬프트·문서에서 이 전제를 어떻게 다루는지 확인 (도메인 감점·규제 리스크)

## 출력

리스크를 심각도순(치명적/주의/참고)으로 정리: `위치(파일:라인) | 리스크 | 근거 | 권고`. 코드를 임의 수정하지 말고 발견 사항만 보고한다.
