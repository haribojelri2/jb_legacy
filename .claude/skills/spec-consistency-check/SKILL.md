# 문서-코드 정합성 검증 하네스

발표 문서와 실제 구현이 어긋나는 지점을 잡아낸다. 심사위원이 코드를 열어 실측과 대조하면 신뢰도 전체가 흔들리므로(과거 실측: 문서 "단위 39건" vs 실제 33건, "검수=GPT-4o-mini" vs 실제 Claude), 이 갭을 사전에 제거한다.

## 검증 절차

1. **테스트 수 실측**
   - 단위 테스트: `python -m pytest tests/ --ignore=tests/test_ui.py --co -q` 의 마지막 줄(`N tests collected`)
   - 화면 자동화: `python -m pytest tests/test_ui.py --co -q` 의 수
   - 문서에서 "단위 테스트 N건", "화면 자동화 N건" 표기를 찾아(`MVP제안서.md`, `기능명세서.md`) 실측과 대조

2. **LLM 모델명**
   - 실제: `agents/llm.py`의 `_DEFAULT_MODELS`와 provider 라우팅 로직
   - 문서: MVP제안서 5-1 기술 구성표, 기능명세서 3-1/6-1의 모델 표기가 실제와 일치하는지

3. **에이전트 목록**
   - 실제: `graph.py`의 `g.add_node(...)` 등록 노드 + `_AGENT_NODE_MAP`
   - 문서: 기능명세서 2 시스템 구성도·3-1 에이전트별 명세의 목록과 대조 (누락·유령 에이전트 확인)

4. **핵심 골든 수치**
   - 권리금 162,000,000 / 매각세 28,754,000(약 2,875만) / 가업승계 특례 0원 등 대표 페르소나(lee_sajang) 수치가 문서·코드·`tests/test_calculators.py`에서 동일한지
   - goal-seek 실측값(A안/B안/C안 지속가능 생활비)이 문서와 일치하는지 — 변동 가능하므로 `eval/run_eval.py` 또는 E2E로 재확인

5. **신기능 반영 여부**
   - 최근 추가한 기능(이중 검증 루프, GAN 적대 검증, goal-seek, monitoring 노드, RAG 청킹)이 문서에 기재됐는지

## 출력

발견한 불일치를 표로: `항목 | 문서 표기 | 실측 | 조치`. 불일치가 없으면 "정합성 OK"로 마무리. 절대 문서를 임의 수정하지 말고, 불일치만 보고한 뒤 사용자 확인 후 고친다(수치는 실측이 진실).
