# 고도화 백로그 하네스

프로젝트의 미완성·개선 여지를 수집해 평가기준에 매핑된 우선순위 백로그를 생성/갱신한다. 이 산출물 자체가 평가기준 4(본선 추가 개발 기반의 충분성)·5(확장성)에 대한 어필이 된다.

## 수집 소스

1. **코드 마커**: `agents/`, `tools/`, `ui/`, `data/`에서 `TODO`, `FIXME`, `DEMO MOCK`, `하드코딩`, `본선` 주석 스캔
2. **기존 고도화 기록**: `OVERVIEW.md`의 고도화 섹션, `기능명세서.md` 5 향후 발전 방향
3. **알려진 부채** (진단에서 도출):
   - 페르소나 4명 하드코딩(USERS dict, 거래내역 lee_sajang 전용) → 온보딩 직접 입력·업종별 파라미터
   - `ui/app.py` 대형 모놀리스 → 컴포넌트 분리
   - C안 JB우리캐피탈 양수도 대출 BM이 문서에만 존재 → 코드화
   - 외부 연동(마이데이터·상권정보) 어댑터 계층
   - 조특법 30조의6 개인사업자 적용요건 룰 체크
   - 배포(FastAPI 분리·Docker/CI)·보안(PII 가명화·감사 로그)

## 우선순위화 기준

각 항목에 부여:
- **평가기준 매핑**: 1(주제적합성) / 2(금융연계·고객가치) / 3(Agent 설계) / 4(MVP 완성도) / 5(혁신·확장·운영리스크) 중 기여하는 항목
- **임팩트**: high / medium / low (심사 어필 + 사용자 가치)
- **노력**: hours / days / weeks
- **지평**: 발표 전(저위험 즉시) / 본선 직후(Agent 심화) / 실서비스(아키텍처)

## 출력

프로젝트 루트에 `IMPROVEMENT_BACKLOG.md`를 생성/갱신:
- 지평별 섹션, 항목당 `제목 / 왜(현재 한계, 파일 근거) / 어떻게 / 평가기준 / 임팩트 / 노력`
- 상단에 "가장 먼저 할 3가지" 요약
- 이미 완료된 항목은 별도 "완료" 섹션으로 이동해 진척을 가시화

임의로 코드를 고치지 않는다 — 백로그 문서만 생성/갱신한다.
