# JB Legacy — 프로그램 전체 설명

## 한 줄 요약

자영업자의 사업 엑시트(매각 or 승계) 의사결정을 AI 멀티에이전트가
**부모 안정 / 가족 자산 지속 / 균형** 세 축으로 분석해서 삶에 가장 적합한 시나리오 하나를 추천하는 시스템.
자녀가 협상 조건을 제안하면 가족 합의안(D안)까지 같은 기준으로 비교한다.

---

## 전체 흐름

```
사용자 질문 입력 (텍스트 또는 음성 STT)
      ↓
[Supervisor] — 에이전트 선택 (LLM 라우팅 + 키워드 결정론 게이트)
      ↓
[Profiler] — 사용자 프로필 + 삶 팩터 주입 (정보 부족 시 추가 질문 후 종료)
      ↓ (병렬 실행 — 4개 fan-out)
┌────────────────────────────────────────────────────────────┐
│ BusinessValuation │ TaxSuccession │ PostExitWM │ Negotiation │
│  사업체 가치 산정  │  세금 비교    │ 노후+생존확률 │ 가족 합의안 D │
└────────────────────────────────────────────────────────────┘
      ↓
[Synthesizer] — 3축 분석 후 A/B/C/D 중 하나 추천 (구조화 출력 강제 + 마크다운 새니타이저)
      ↓
[SlowUIAdapter] — 이사장용 큰 글씨·쉬운 말 변환
      ↓
[ComplianceGuard] — 불완전판매 문구 검수 (미통과 시 재생성, 최대 3회)
      ↓
[GANReview] — 적대 검증: Critic·Defender·Judge 토론 채점 (GAN_AUTO=1 시 실행,
      '재생성필요' 판정이면 개선 지시 주입 후 Synthesizer 재생성 1회)
      ↓
[FamilyBridge] — 딸(이과장)에게 공유 (선택)
      ↓
[Booking] — PB 상담 예약 (선택)
```

그래프 밖 UI 계층 기능: 거래 안심(FraudGuard — 이상거래 탐지·가족 알림), 조기경보,
권리금 동적 평가, 청년 매칭, 자문료 계약 플랜, PDF 리포트, TTS 음성 브리핑, what-if 전후 비교,
AI 검증 탭(GAN 수동 채점 + 개선 반영 재생성 — 전후 점수 비교, 실측 68→78점 상승 확인).

## 시나리오 4가지의 의미

산출 조건: 이사장 기준 — 승계 의향 예 / 상권 보합 / 은퇴 3년 이내 / 목표 생활비 월 300만 / 주택연금 활용(월 66.3만)

| | A. 완전 매각 | B. 완전 승계 | C. 절충 | D. 가족 합의안 |
|---|---|---|---|---|
| 최적화 대상 | 부모 노후 안정 | 가족 자산 지속 | 균형 (금융 연계형 승계) | 협상 (자녀 제안 조건) |
| 부모 월수령 | 약 288만 | 약 289만 | 약 282만 | 약 292만 (70%/20% 기준) |
| 가족 총자산 10년 후 | 포트폴리오만 | 사업 지속 가치 포함 | 절반씩 | 승계 비율만큼 |
| 브랜드·고용 유지 | 소멸 | 유지 | 가족 내 유지 | 유지 |

A가 "더 많은 현금"이고 B가 "손해"가 아니라, **다른 가치를 최적화**하는 선택임.

> **현실적인 C안(절충안)의 비즈니스 모델: "JB우리캐피탈 승계 자금 대출 연계"**
> 현실의 개인사업자 단위에서 "가게 지분의 절반만 모르는 사람에게 파는 것"은 매우 어렵습니다. 따라서 본 프로젝트에서 C안은 **"자녀가 JB우리캐피탈(또는 전북은행)에서 양수도 자금 대출을 받아, 부모에게 권리금 절반을 현금으로 지급하고 가업을 승계받는 모델"**로 정의합니다.
> - **부모 (안정)**: 자녀가 융통해 준 대출금으로 즉시연금/펀드에 가입해 노후 현금(목돈) 확보
> - **자녀 (지속)**: 가업을 온전히 승계받고, 가게 수익으로 대출 원리금 상환
> - **JB금융그룹 (수익)**: 여신(JB우리캐피탈 대출 이자)과 수신(전북은행/JB자산운용 예금·펀드)을 동시에 창출하는 **완벽한 양방향 크로스셀링(Cross-selling)** 달성

## 에이전트별 역할

### Supervisor
- GPT-4o-mini로 질문 분석 후 실행할 에이전트 목록 결정
- 핵심 3개(BusinessValuation·TaxSuccession·PostExitWM)가 하나도 선택되지 않으면 전부 기본 투입
  (단독 주제 질문이면 해당 에이전트 1개만 실행될 수 있음 — synthesizer가 단일 응답 경로 처리)
- FamilyBridge는 "딸, 공유, 알려줘" 키워드, Negotiation은 "협상, 합의안, 승계+제안" 키워드 시
  **LLM 판단과 무관하게 결정론 규칙으로 추가** (데모 라우팅 안정성)

### Profiler
- mock_data에서 사용자 프로필 로드
- UI 드롭다운 5개를 `life_factors`로 병합하여 이후 에이전트에 전달
- 필수 재무 정보(권리금·월순이익·저축) 없으면 추가 질문 반환

### BusinessValuation
- `estimate_business_value(월순이익, 운영연수)`로 권리금 산정
  - 20년 이상 → 36개월 배수 (우량 A등급)
  - 10~20년 → 30개월 배수 (안정 B등급)
  - 10년 미만 → 24개월 배수 (신생 C등급)
- 총 사업체 가치 = 권리금 + 보증금 + 시설·집기

### TaxSuccession
- **외부 매각**: 권리금 × 40% 과세표준 + 연간 사업소득 합산 → 종합소득세(기타소득)
- **가업승계 과세특례** (조특법 제30조의6): 사업체가치 - 10억 공제 후 10% 증여세
- **일반 증여**: 성인 자녀 5,000만 공제 후 일반 증여세율
- RAG로 관련 세법 문서 검색 후 GPT-4o 설명 생성
- 계산용·표시용 세율표는 `tools/calculators.py` 단일 출처 (`get_tax_bracket`)

### PostExitWM (핵심)

TaxSuccession·BusinessValuation과 병렬 실행 → 결과를 읽지 못하므로 **동일 함수 직접 호출**:

```
권리금    = estimate_business_value() 직접 호출
매각 세금 = calc_goodwill_tax(권리금, other_income=연소득) 직접 계산
가업 가치 = calc_business_continuity(월순이익, 운영연수, 상권 트렌드) 직접 계산
```

**시나리오 3개 계산**

| | A. 완전 매각 | B. 완전 승계 | C. 절충 |
|---|---|---|---|
| 운용자산 | 사업체가치 - 매각세금 + 저축 | 저축만 | 권리금 50% - 세금 + 저축 |
| 자문료 | 없음 | 월순이익 × 20% (10년) | 월순이익 × 10% (10년) |
| 가업 지속 가치 | 없음 | 전체 반영 | 50% 반영 |
| 승계 의향 아니오 | 계산 | 생략 | 생략 |

**포트폴리오 배분** — 전부 전북은행·광주은행 실존 상품, 목표 생활비 기반 **동적 배분**

| 상품 | 역할 | 방식 | 만료 |
|---|---|---|---|
| JB 리치 100 정기예금 (즉시연금형) · 전북은행 | 월 현금흐름 핵심 | 매월 원금+이자 분할 수령 (이자분 15.4% 과세) | 10년 |
| KJ 정기예금 · 광주은행 | 원금 보존 | 이자만 수령 (예금자보호 1억) | 없음 |
| JB 개인형 퇴직연금(IRP) · 전북은행 | 세액공제 연금 | (85세 - 나이)년 분할 수령 | 나이 연동 |

**동적 배분 규칙**: IRP 15% 고정(세액공제), 나머지 85%를 즉시연금형↔예금으로 목표 생활비에 맞춰 조절.
목표가 높으면(빠듯) 즉시연금형↑(원금까지 활용해 월수령 극대화), 여유로우면 예금↑(원금 보존).
예) 목표 250만 → 즉시연금형 12%/예금 72%, 목표 400만 → 즉시연금형 45%/예금 39%.
모든 수령액은 **세후** 기준. (JB금융그룹에 생명보험 계열사가 없어 즉시연금은 보험이 아닌 전북은행 정기예금 즉시연금형으로 제공 — 사업비 없음.)

**은퇴 자산 생존 확률** (`tools/monte_carlo.py`)

- 모형: 월 수익률 정규분포 + 긴급 의료비 복합 포아송 과정(CPP — 연 0.25회, 평균 3,000만 지수분포)
- 1,000회 경로 × 100세까지, 시드 고정(42), 시나리오 간 동일 난수(CRN) 통제 비교
- 순인출 = 목표 생활비 - (국민연금 + 주택연금 + 자문료), 자문료 10년 한정
- 수익률 가정: 전부 예금/연금형 실제 상품 기준 저위험 — A 3.3%/1.2%, B 3.0%/1.0%, C 3.1%/1.1%
- 예금 수익률로는 목표 생활비를 38년간 충당하기 어려워 A·B·C 모두 생존확률이 낮게(0%대) 나옴 — "매각 후 예금만으로는 부족하니 주택연금·지출조정·승계가 중요"라는 근거로 활용
- 출력: 연령별 생존 확률 곡선, 고갈 연령 구간 분포, 잔여 자산 중앙값·하위 10%

**추가 소득원 (선택)**

- 주택연금: 한국주택금융공사 공시 기준, 종신지급 정액형
  - 62세·3억 기준 → 월 663,000원, 사망까지 지급 (만료 없음)
  - 집에 계속 거주 가능, A·B·C 모든 시나리오 동일 적용 (주택은 개인 자산)
- 국민연금: 고정 입력값, 장기 시뮬레이션에서 물가 연동 증가 반영

**장기 재정 시뮬레이션** (0·5·10·15·20년 시점)

- 명목 월수령액 / 실질 구매력(물가 2%) / 잔여 원금
- 즉시연금 10년 만료·IRP N년 만료 시점 자동 반영

**가업 지속 가치** (`calc_business_continuity`)

상권 트렌드별 성장률로 B·C 시나리오의 자녀 관점 미래가치 계산:

| 상권 | 연 성장률 | 딸 10년 누적 수익 | 10년 후 권리금 | 가족 총자산 증가 |
|---|---|---|---|---|
| 성장 | +5% | 5억7,000만 | 2억6,400만 | 8억3,400만 |
| 보합 | +1% | 4억5,600만 | 1억7,900만 | 6억3,500만 |
| 하락 | -3% | 3억6,700만 | 1억1,900만 | 4억8,600만 |

(이사장 데이터 기준, B안 전체 승계, 자문료 20% 차감 후)

### Negotiation (가족 협상 — D안)

- 본 분석 파이프라인의 4번째 병렬 노드. 두 가지 진입 경로:
  1. 자녀 화면 협상 폼 (승계 비율·자문료율 슬라이더 + 한마디)
  2. 부모 채팅 ("딸이 70% 승계에 자문료 20%를 제안하면?") — 질문에서 조건 자동 추출, 실패 시 기본 70%/20%
- 결정론 게이트: 협상 키워드 또는 협상 입력이 있을 때만 동작 (LLM 과잉 라우팅 차단)
- D안 = 부모 현금화 (1-승계율)×권리금 - 세금 + 저축 → 포트폴리오, 자문료 = 월순이익×자문료율(10년)
- D안 결과는 Synthesizer 종합 권고와 ComplianceGuard 검수를 동일하게 거침

### Synthesizer

- 4개 에이전트 의견 + 시나리오 수치 + 장기 시뮬레이션 + 가업 지속 가치 + 삶 팩터를 GPT-4o에 주입
- **3축 분석 프레임**으로 판단:
  1. 부모 안정축 (A 강점): 월수령·20년 실질 구매력·원금 소진 시점
  2. 가족 자산 지속축 (B 강점): 딸 누적 수익·10년 후 재매각가치·가족 총자산
  3. 균형축 (C 강점): 부모 현금 + 가업 일부 지속 절충 효과
  4. (D안 존재 시) 가족 합의축 추가
- **반드시 A·B·C·D 중 정확히 하나만 추천** — `Literal` 구조화 출력으로 강제
- [계산 근거] 섹션은 LLM이 아니라 코드가 직접 조립해 삽입 (수치 환각 차단)

### ComplianceGuard
- 투자원금 보장, 확정 수익률 등 불완전판매 문구 감지
- 위반 시 지적사항을 피드백하여 Synthesizer 재실행 (최대 3회, 초과 시 PB 연결 전환)

### SlowUIAdapter
- 이사장(slow_ui=True): 큰 글씨·쉬운 말·존댓말 변환
- 원본은 `final_response_raw` 보존 → 이과장 화면·PDF 리포트에서 원본 사용

### FamilyBridge / Booking
- FamilyBridge: 딸에게 공유할 메시지 생성
- Booking: 가까운 JB 지점 PB 상담 예약 확인서 생성

### FraudGuard (UI 계층 — 거래 안심)
- 최근 90일 거래내역에서 룰 4종으로 이상거래 감지 (LLM 미사용, 결정론):
  고액 이탈(평균+3σ, 100만 초과) / 심야(00~06시) / 동일 가맹점 분할 결제(합계 200만 초과) / 해외
- 복수 사유 = 위험도 높음. 소비흐름(카테고리별 월평균 지출) 분석 병행
- 감지 시 자녀에게 지급정지 안내 포함 알림 발송 (FamilyBridge 연계 스토리)
- 시드 데이터 기준 미탐 0·오탐 0을 단위 테스트로 고정

- 전부 전북은행·광주은행 실제 예금/연금형 상품 기준(세후). 펀드·보험을 빼고 예금 위주로 구성해 수익률이 낮아져, 월수령이 목표(300만)에 약간 못 미치고 생존확률도 0%대로 나옴.
