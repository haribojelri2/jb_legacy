# JB Legacy — 프로그램 전체 설명

## 한 줄 요약

자영업자의 사업 엑시트(매각 or 승계) 의사결정을 AI 멀티에이전트가  
**부모 안정 / 가족 자산 지속 / 균형** 세 축으로 분석해서 삶에 가장 적합한 시나리오 하나를 추천하는 시스템

---

## 전체 흐름

```
사용자 질문 입력
      ↓
[Supervisor] — 에이전트 선택
      ↓
[Profiler] — 사용자 프로필 + 삶 팩터 주입
      ↓ (병렬 실행)
┌──────────────────────────────────────────────┐
│ BusinessValuation │ TaxSuccession │ PostExitWM │
│  사업체 가치 산정  │  세금 비교    │ 노후+가업가치 │
└──────────────────────────────────────────────┘
      ↓
[Synthesizer] — 3축 분석 후 시나리오 하나 추천
      ↓
[SlowUIAdapter] — 이사장용 큰 글씨·쉬운 말 변환
      ↓
[ComplianceGuard] — 불완전판매 문구 검수 (최대 3회 재시도)
      ↓
[FamilyBridge] — 딸(이과장)에게 공유 (선택)
      ↓
[Booking] — PB 상담 예약 (선택)
```

---

## 시나리오 3가지의 의미

| | A. 완전 매각 | B. 완전 승계 | C. 절충 |
|---|---|---|---|
| 최적화 대상 | 부모 노후 안정 | 가족 자산 지속 | 균형 (금융 연계형 승계) |
| 부모 월수령 | 최대 (307만) | 최소 (200만) | 중간 (246만) |
| 가족 총자산 10년 후 | 포트폴리오만 | 사업 지속 가치 포함 | 절반씩 |
| 브랜드·고용 유지 | 소멸 | 유지 | 가족 내 유지 |

A가 "더 많은 현금"이고 B가 "손해"가 아니라, **다른 가치를 최적화**하는 선택임.

> **💡 현실적인 C안(절충안)의 비즈니스 모델: "JB우리캐피탈 승계 자금 대출 연계"**
> 현실의 개인사업자 단위에서 "가게 지분의 절반만 모르는 사람에게 파는 것"은 매우 어렵습니다. 따라서 본 프로젝트에서 C안은 **"자녀가 JB우리캐피탈(또는 전북은행)에서 양수도 자금 대출을 받아, 부모에게 권리금 절반을 현금으로 지급하고 가업을 승계받는 모델"**로 정의합니다.
> - **부모 (안정)**: 자녀가 융통해 준 대출금으로 즉시연금/펀드에 가입해 노후 현금(목돈) 확보
> - **자녀 (지속)**: 가업을 온전히 승계받고, 가게 수익으로 대출 원리금 상환
> - **JB금융그룹 (수익)**: 여신(JB우리캐피탈 대출 이자)과 수신(전북은행/JB자산운용 예금·펀드)을 동시에 창출하는 **완벽한 양방향 크로스셀링(Cross-selling)** 달성

## 에이전트별 역할

### Supervisor
- GPT-4o-mini로 질문 분석 후 실행할 에이전트 목록 결정
- BusinessValuation·TaxSuccession·PostExitWM은 **항상 3개 동시 실행**
  (질문 방식마다 추천이 달라지는 버그 방지 — 세 에이전트는 서로 입력값 공유)
- FamilyBridge는 "딸, 공유, 알려줘" 키워드 있을 때만 추가

### Profiler
- mock_data에서 사용자 프로필 로드
- UI 드롭다운 5개를 `life_factors`로 병합하여 이후 에이전트에 전달
- 필수 재무 정보(권리금·월순이익·저축) 없으면 추가 질문 반환

### BusinessValuation
- `estimate_business_value(월순이익, 운영연수)`로 권리금 산정
  - 20년 이상 → 36개월 배수 (우량 A등급)
  - 10~20년 → 30개월 배수 (안정 B등급)
  - 10년 미만 → 24개월 배수 (신생 C등급)
- 총 사업체 가치 = 권리금 + 보증금 + 시설·집기

### TaxSuccession
- **외부 매각**: 권리금 × 40% 과세표준 + 연간 사업소득 합산 → 종합소득세(기타소득)
- **가업승계 과세특례** (조특법 제30조의6): 사업체가치 - 10억 공제 후 10% 증여세
- **일반 증여**: 성인 자녀 5,000만 공제 후 일반 증여세율
- RAG로 관련 세법 문서 검색 후 GPT-4o 설명 생성

### PostExitWM (핵심)

TaxSuccession·BusinessValuation과 병렬 실행 → 결과를 읽지 못하므로 **동일 함수 직접 호출**:

```
권리금    = estimate_business_value() 직접 호출
매각 세금 = calc_goodwill_tax(권리금, other_income=연소득) 직접 계산
가업 가치 = calc_business_continuity(월순이익, 운영연수, 상권 트렌드) 직접 계산
```

**시나리오 3개 계산**

| | A. 완전 매각 | B. 완전 승계 | C. 절충 |
|---|---|---|---|
| 운용자산 | 사업체가치 - 매각세금 + 저축 | 저축만 | 권리금 50% - 세금 + 저축 |
| 자문료 | 없음 | 월순이익 × 20% (10년) | 월순이익 × 10% (10년) |
| 가업 지속 가치 | 없음 | 전체 반영 | 50% 반영 |
| 승계 의향 아니오 | 계산 | 생략 | 생략 |

**포트폴리오 배분** (운용자산 1억 이상)

| 상품 | 비중 | 방식 | 만료 |
|---|---|---|---|
| JB 월배당 혼합채권형 펀드 | 35% | 배당수익만 수령 (원금 보존) | 없음 |
| JB 즉시연금보험 | 25% | 사업비 7% 차감 후 10년 PMT | 10년 |
| KJ 정기예금 | 25% | 이자수익만 수령 (원금 보존) | 없음 |
| JB IRP | 15% | (85세 - 나이)년 PMT | 나이 연동 |

운용자산 1억 미만이면 즉시연금 단일 상품 집중.  
모든 수령액은 **세후** 기준 (이자·배당 15.4%, 연금소득 5.5% 차감).

**추가 소득원 (선택)**

- 주택연금: 한국주택금융공사 공시 기준, 종신지급 정액형
  - 62세·3억 기준 → 월 663,000원, 사망까지 지급 (만료 없음)
  - 집에 계속 거주 가능, A·B·C 모든 시나리오 동일 적용 (주택은 개인 자산)
- 국민연금: 고정 입력값, 장기 시뮬레이션에서 물가 연동 증가 반영

**장기 재정 시뮬레이션** (0·5·10·15·20년 시점)

- 명목 월수령액 / 실질 구매력(물가 2%) / 잔여 원금
- 즉시연금 10년 만료·IRP N년 만료 시점 자동 반영

**가업 지속 가치** (`calc_business_continuity`)

상권 트렌드별 성장률로 B·C 시나리오의 자녀 관점 미래가치 계산:

| 상권 | 연 성장률 | 딸 10년 누적 수익 | 10년 후 권리금 | 가족 총자산 증가 |
|---|---|---|---|---|
| 성장 | +5% | 5억7,000만 | 2억6,400만 | 8억3,400만 |
| 보합 | +1% | 4억5,600만 | 1억7,900만 | 6억3,500만 |
| 하락 | -3% | 3억6,700만 | 1억1,900만 | 4억8,600만 |

(이사장 데이터 기준, B안 전체 승계, 자문료 20% 차감 후)

### Synthesizer

- 3개 에이전트 의견 + 시나리오 수치 + 장기 시뮬레이션 + 가업 지속 가치 + 삶 팩터를 GPT-4o에 주입
- **3축 분석 프레임**으로 판단:
  1. 부모 안정축 (A 강점): 월수령·20년 실질 구매력·원금 소진 시점
  2. 가족 자산 지속축 (B 강점): 딸 누적 수익·10년 후 재매각가치·가족 총자산
  3. 균형축 (C 강점): 부모 현금 + 가업 일부 지속 절충 효과
- **반드시 A·B·C 중 정확히 하나만 추천** (복수 추천·"상황에 따라" 금지)
- 추천 이유: "부모 안정 vs 가족 자산 지속 중 이 분 상황에 더 중요한 것" 기준으로 설명

### ComplianceGuard
- 투자원금 보장, 확정 수익률 등 불완전판매 문구 감지
- 위반 시 Synthesizer 재실행 (최대 3회)

### SlowUIAdapter
- 이사장(slow_ui=True): 큰 글씨·쉬운 말·존댓말 변환
- 원본은 `final_response_raw` 보존 → 이과장 화면에서 원본 사용

### FamilyBridge / Booking
- FamilyBridge: 딸에게 공유할 메시지 생성
- Booking: 가까운 JB 지점 PB 상담 예약 확인서 생성

## 사용자 입력 5가지 (드롭다운)

| 항목 | 선택지 | 영향 |
|---|---|---|
| 따님 승계 의향 | 예 / 아니오 | 아니오 → B·C 시나리오 완전 생략 |
| 지역 상권 트렌드 | 성장 / 보합 / 하락 | 가업 지속 가치 성장률 + Synthesizer 추천 근거 |
| 은퇴 시점 | 1년 이내 / 3년 이내 / 5년 이상 | Synthesizer 추천 근거 |
| 월 목표 생활비 | 150만 ~ 400만원 | 시나리오별 목표 달성 여부 기준선 |
| 주택연금 활용 | 예 / 아니오 | 예 → 월 663,000원 추가 (3억·62세 기준) |

## 현재 데모 페르소나

### 이사장 (62세, 전북 전주)
- 이씨 한정식 30년 운영
- 월순이익 450만원, 월매출 1,200만원
- 권리금 1억6,200만 + 보증금 1억 + 시설 5,000만 = **사업체 3억1,200만원**
- 개인 저축 5,000만원 / 국민연금 월 90만원 / 거주용 아파트 3억원

### 이과장 (32세, 서울, 딸)
- 자녀 앱에서 아버지 은퇴 플랜을 공유 받는 역할
- Split-Screen 모드 또는 "딸에게 공유" 버튼으로 전환

|---|---|---|---|---|---|
| A. 완전 매각 | 3억3,300만 | 2,875만 | 241만원 | **307만원** | **+7만 ✓** |
| B. 완전 승계 | 5,000만 | — | 134만원 | 200만원 | -100만 |
| C. 절충안 | ~1억5,000만 | ~1,200만 | 180만원 | 246만원 | -54만 |

10년 후 즉시연금 만료 시 A안 실질 구매력: 약 212만원 (주택연금 포함)

### 가족 관점 (상권 보합 기준, B안 전체 승계)

## 하드코딩 → 실제 서비스 시 변수화 필요 항목

| 항목 | 현재 값 | 변수화 방법 |
|---|---|---|
| `target_monthly` | 드롭다운 150~400만 | 온보딩 폼 직접 입력으로 전환 |
| JB 상품 금리 | `data/jb_products.py` 파일 | JB 상품 API 또는 백오피스 DB 연동 |
| `_ANNUITY_EXPENSE_RATIO` | 7% 고정 | 가입 상품 약관값으로 대체 |
| `_LONGEVITY_AGE` | 85세 고정 | 통계청 기대여명 테이블 연동 |
| `_INFLATION_RATE` | 2% 고정 | 한국은행 전망치 연동 |
| `_HOME_PENSION_PER_100M` | 공시 테이블 하드코딩 | HF공사 API 연동 |
| 가업 성장률 (성장+5%/보합+1%/하락-3%) | `calc_business_continuity` 상수 | 업종별 통계청 데이터 연동 |
| 세율 구간 | `tools/calculators.py` | 세법 개정 시 수동 업데이트 |
| 사용자 프로필 | `data/mock_data.py` | CRM 또는 온보딩 폼 → DB 조회 |

## 파일 구조

```
jb_legacy/
├── graph.py                  # LangGraph 오케스트레이터, run_query() 진입점
├── agents/
│   ├── state.py              # AgentState 타입 정의
│   ├── supervisor.py         # 에이전트 라우팅 (3개 핵심 에이전트 항상 실행)
│   ├── profiler.py           # 프로필 + 삶 팩터 주입
│   ├── business_valuation.py # 사업체 가치 산정
│   ├── tax_succession.py     # 세금 비교 (RAG 포함)
│   ├── post_exit_wm.py       # 노후 포트폴리오 + 가업 지속 가치 계산
│   ├── synthesizer.py        # 3축 분석 후 최종 추천
│   ├── slow_ui_adapter.py    # 시니어 UI 변환
│   ├── compliance.py         # 불완전판매 검수
│   ├── family_bridge.py      # 자녀 공유
│   └── booking.py            # PB 예약
├── tools/
│   └── calculators.py        # 세금·가치평가·포트폴리오·가업지속가치 계산 함수
├── data/
│   ├── mock_data.py          # 데모 페르소나 데이터
│   └── jb_products.py        # JB금융그룹 상품 카탈로그
├── rag/
│   └── retriever.py          # 세법 문서 벡터 검색
├── ui/
│   └── app.py                # Streamlit UI
└── OVERVIEW.md               # 이 파일
```
